2.2 CriteriaBuilderFactory 初始化

CriteriaBuilderFactory 是 Blaze-Persistence 的查询入口—— 所有 CriteriaBuilder 实例都从它创建。 它的初始化遵循 Configuration → Factory 模式：先通过可变的配置对象注册属性， 然后调用 createCriteriaBuilderFactory() 生成不可变的工厂实例。

生命周期

图 2-3 · CriteriaBuilderFactory 初始化时序图
sequenceDiagram participant App as 应用启动 participant EMF as EntityManagerFactory participant Cfg as CriteriaBuilderConfiguration participant SL as ServiceLoader participant CBF as CriteriaBuilderFactory App->>EMF: Persistence.createEntityManagerFactory() activate EMF EMF-->>App: emf 实例 deactivate EMF App->>Cfg: Criteria.getDefault() activate Cfg Cfg->>SL: 加载 CriteriaBuilderConfigurationProvider SL-->>Cfg: 默认实现 Cfg-->>App: configuration 实例 deactivate Cfg App->>Cfg: setProperty("key", "value") App->>Cfg: registerFunction(...) App->>Cfg: createCriteriaBuilderFactory(emf) activate Cfg Cfg->>Cfg: 验证配置 + 解析 JPA 元模型 Cfg->>CBF: 构建不可变工厂 CBF-->>App: cbf 实例（线程安全） deactivate Cfg Note over CBF: 此后 CBF 不可修改
可在多线程间共享
💡 线程安全保证 CriteriaBuilderFactory 创建后是不可变的（immutable）， 可以安全地作为单例存储在 Spring 容器或静态字段中，供任意线程并发调用。

代码示例：手动初始化

// 1. 获取 JPA EntityManagerFactory（通常来自 persistence.xml）
EntityManagerFactory emf =
    Persistence.createEntityManagerFactory("my-pu");

// 2. 创建 CriteriaBuilderConfiguration
CriteriaBuilderConfiguration config =
    Criteria.getDefault();

// 3. 可选：注册自定义配置
config.setProperty(
    ConfigurationProperties.COMPATIBLE_MODE,
    "false"
);

// 4. 构建不可变工厂
CriteriaBuilderFactory cbf =
    config.createCriteriaBuilderFactory(emf);

// 5. 使用 —— 从 cbf 创建查询
CriteriaBuilder<Cat> cb = cbf.create(em, Cat.class);
List<Cat> cats = cb.where("age").gt(3).getResultList();

两种注册方式对比

📌 手动逐个注册
EntityViewConfiguration cfg =
    EntityViews.createDefaultConfiguration();

cfg.addEntityView(CatView.class);
cfg.addEntityView(PersonView.class);
cfg.addEntityView(OrderSummary.class);

使用 @Qualifier 区分多个工厂 Bean：

@Configuration
public class MultiDataSourceBlazeConfig {

    @Bean @Qualifier("order")
    public CriteriaBuilderFactory orderCbf(
            @Qualifier("order") EntityManagerFactory emf) {
        return Criteria.getDefault()
            .createCriteriaBuilderFactory(emf);
    }

    @Bean @Qualifier("user")
    public CriteriaBuilderFactory userCbf(
            @Qualifier("user") EntityManagerFactory emf) {
        return Criteria.getDefault()
            .createCriteriaBuilderFactory(emf);
    }

    @Bean @Qualifier("order")
    public EntityViewManager orderEvm(
            @Qualifier("order") CriteriaBuilderFactory cbf) {
        EntityViewConfiguration cfg =
            EntityViews.createDefaultConfiguration();
        cfg.addEntityViewPackage("com.demo.order.view");
        return cfg.createEntityViewManager(cbf);
    }

    @Bean @Qualifier("user")
    public EntityViewManager userEvm(
            @Qualifier("user") CriteriaBuilderFactory cbf) {
        EntityViewConfiguration cfg =
            EntityViews.createDefaultConfiguration();
        cfg.addEntityViewPackage("com.demo.user.view");
        return cfg.createEntityViewManager(cbf);
    }
}
⚠️ Entity View 不能跨工厂共享 一个 EntityViewManager 只认识它注册时所绑定的 CriteriaBuilderFactory。 跨数据源使用错误的 EVM 会导致运行时异常。

Blaze-Persistence 对 Hibernate 版本有严格要求，使用错误的集成模块会触发 NoSuchMethodError。

陷阱 3：Spring Bean 初始化顺序

CriteriaBuilderFactory 依赖 EntityManagerFactory， EntityViewManager 依赖 CriteriaBuilderFactory。 如果使用 @Lazy 或条件装配，可能打断依赖链。

1
EntityManagerFactory 必须最先就绪（由 Spring JPA 自动配置保证）
2
CriteriaBuilderFactory 在 EMF 之后创建，标记 @Lazy(false)
3
EntityViewManager 在 CBF 之后创建，同样 @Lazy(false)
4
业务 Service / Repository 最后注入 CBF 和 EVM

陷阱 4：Spring Boot Starter 版本选择

Starter 模块的命名从 1.6.9 开始区分 Spring Boot 2.x 和 3.x。 使用错误版本将导致自动配置完全失效但不报错。

Spring BootStarter artifactId
2.7.xblaze-persistence-integration-spring-boot-starter
3.0 ~ 3.3blaze-persistence-integration-spring-boot-3.1-starter

📝 本章小结

  • Blaze-Persistence 采用模块化设计，通过 BOM 统一版本，API 与实现分离
  • CriteriaBuilderFactory 是查询入口，创建后不可变且线程安全
  • EntityViewManager 负责 DTO 投影，需在所有 View 注册后一次性构建
  • Spring Boot Starter 将 ~25 行配置代码缩减为 2 行依赖 + yml
  • 多数据源场景需要为每个 DataSource 创建独立的工厂链
  • 最常见的 5 类陷阱：版本不匹配、View 未注册、Bean 顺序、Starter 版本选错、scope 误用
