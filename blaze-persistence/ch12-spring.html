12.2 EntityViewRepository

EntityViewRepository<V, ID> 是 Blaze-Persistence 对 Spring Data JpaRepository 的等价替换。 它不操作实体类 (Entity)，而是直接以 Entity View 作为投影载体进行 CRUD，从而确保所有查询只 SELECT 视图声明的字段。

12.3 方法名查询派生

Spring Data 的方法名约定（Query Derivation）在 Blaze-Persistence 集成中同样有效。框架会将方法名解析为等价的 CriteriaBuilder 查询，而非原生 JPQL，从而享受类型安全和视图投影优化。

💡 关键洞察：与原生 Spring Data JPA 不同，Blaze-Persistence 的查询派生通过 CriteriaBuilder 构建查询，而非直接拼接 JPQL 字符串。这意味着在多表关联场景中，JOIN 策略由框架自动优化，不会产生不必要的 CROSS JOIN。

12.4 EntityViewSpecification

当查询条件是运行时动态组合的（例如后台管理界面的多条件筛选），方法名派生就力不从心了。EntityViewSpecificationExecutor 提供了 Specification 模式，将过滤逻辑封装为可组合的谓词对象。

定义 Specification

public class UserViewSpecs {

    public static EntityViewSpecification<UserView> nameLike(String fragment) {
        return (root, cb) ->
            cb.where(root.get("name")).like().value("%" + fragment + "%");
    }

    public static EntityViewSpecification<UserView> statusEq(UserStatus status) {
        return (root, cb) ->
            cb.where(root.get("status")).eq(status);
    }

    public static EntityViewSpecification<UserView> ageGreaterThanOrEqual(int min) {
        return (root, cb) ->
            cb.where(root.get("age")).ge(min);
    }
}

在 Controller 中组合 Specification

@GetMapping("/users")
public Page<UserView> searchUsers(
        @RequestParam(required = false) String name,
        @RequestParam(required = false) UserStatus status,
        @RequestParam(required = false) Integer minAge,
        Pageable pageable) {

    EntityViewSpecification<UserView> spec = EntityViewSpecification.where(null);

    if (name != null)   spec = spec.and(UserViewSpecs.nameLike(name));
    if (status != null) spec = spec.and(UserViewSpecs.statusEq(status));
    if (minAge != null) spec = spec.and(UserViewSpecs.ageGreaterThanOrEqual(minAge));

    return userViewRepository.findAll(spec, pageable);
}

12.5 分页与排序集成

Blaze-Persistence 最强大的分页能力是 Keyset 分页（见第 10 章）。在 Spring Data 集成中，你可以透明地将 Pageable / Sort 与 Keyset 分页结合，获得 O(1) 分页性能的同时保留 Spring 生态的兼容性。

REST → Keyset 分页完整请求链路
sequenceDiagram participant Client as 前端 Client participant Ctrl as Controller participant Repo as EntityViewRepository participant CB as CriteriaBuilder participant DB as 数据库 Client->>Ctrl: GET /orders?page=0&size=20&sort=createdAt,desc Ctrl->>Ctrl: 构建 KeysetPageable Ctrl->>Repo: findAll(spec, keysetPageable) Repo->>CB: applySetting(keysetPageable) CB->>DB: SELECT ... WHERE (createdAt, id) < (?, ?)
ORDER BY createdAt DESC, id DESC
LIMIT 21 DB-->>CB: 21 rows (20 + 1 探测行) CB-->>Repo: KeysetAwarePage<OrderView> Repo-->>Ctrl: KeysetAwarePage Ctrl-->>Client: { content: [...], keyset: { lowest: {...}, highest: {...} } } Note over Client, DB: 下一页请求携带 keyset 令牌，无需 OFFSET

Controller 分页代码

@RestController
@RequestMapping("/api/orders")
public class OrderController {

    private final OrderViewRepository repository;

    @GetMapping
    public KeysetAwarePage<OrderSummaryView> list(
            @RequestParam(required = false) String status,
            @KeysetConfig(keysetClass = OrderSummaryView.class)
            KeysetPageable pageable) {

        EntityViewSpecification<OrderSummaryView> spec =
            status != null ? OrderViewSpecs.statusEq(status) : null;

        return repository.findAll(spec, pageable);
    }
}

12.6 事务管理

Entity View 的保存操作（save()、saveFull()）会触发底层 EntityManager 的 flush，因此必须运行在活跃事务中。Spring 的 @Transactional 注解与 Entity View CRUD 天然协同。

事务边界与 Entity View 保存操作时序
sequenceDiagram participant Svc as @Service participant TX as TransactionManager participant Repo as EntityViewRepository participant EVM as EntityViewManager participant EM as EntityManager participant DB as 数据库 Svc->>TX: @Transactional 开始 TX->>TX: 开启事务 BEGIN Svc->>Repo: save(updatableView) Repo->>EVM: FlushOperationBuilder EVM->>EVM: 视图脏检查
对比快照 EVM->>EM: merge() / persist() EM->>DB: INSERT / UPDATE SQL Svc->>Repo: findById(id) Repo->>EVM: applySetting(CriteriaBuilder) EVM->>DB: SELECT 视图字段 TX->>DB: COMMIT TX-->>Svc: 事务完成

Service 层事务示例

@Service
@Transactional
public class UserService {

    private final UserViewRepository viewRepo;
    private final EntityViewManager evm;

    public UserService(UserViewRepository viewRepo, EntityViewManager evm) {
        this.viewRepo = viewRepo;
        this.evm = evm;
    }

    public void updateUserName(Long userId, String newName) {
        UpdatableUserView user = viewRepo.findById(userId)
                .orElseThrow(() -> new NotFoundException(userId));
        user.setName(newName);
        viewRepo.save(user);
    }

    @Transactional(readOnly = true)
    public List<UserView> listActiveUsers() {
        return viewRepo.findByActiveTrue();
    }

    public UpdatableUserView createUser(String name, String email) {
        UpdatableUserView user = evm.create(UpdatableUserView.class);
        user.setName(name);
        user.setEmail(email);
        return viewRepo.save(user);
    }
}

测试配置类

@TestConfiguration
public class BlazePersistenceTestConfig {

    @Bean
    @Scope(ConfigurableBeanFactory.SCOPE_SINGLETON)
    @Lazy(false)
    public CriteriaBuilderFactory criteriaBuilderFactory(EntityManagerFactory emf) {
        CriteriaBuilderConfiguration config = Criteria.getDefault();
        return config.createCriteriaBuilderFactory(emf);
    }

    @Bean
    @Scope(ConfigurableBeanFactory.SCOPE_SINGLETON)
    @Lazy(false)
    public EntityViewManager entityViewManager(
            CriteriaBuilderFactory cbf,
            EntityViewConfiguration evc) {
        return evc.createEntityViewManager(cbf);
    }

    @Bean
    public EntityViewConfiguration entityViewConfiguration() {
        EntityViewConfiguration cfg = EntityViews.createDefaultConfiguration();
        cfg.addEntityView(UserView.class);
        cfg.addEntityView(UpdatableUserView.class);
        cfg.addEntityView(OrderSummaryView.class);
        return cfg;
    }
}

集成测试示例

@DataJpaTest
@Import(BlazePersistenceTestConfig.class)
@AutoConfigureTestDatabase(replace = Replace.NONE)
class UserViewRepositoryTest {

    @Autowired
    private UserViewRepository repository;

    @Autowired
    private TestEntityManager tem;

    @Test
    void shouldFindByNameContaining() {
        tem.persist(new User("Alice", "alice@test.com", UserStatus.ACTIVE));
        tem.persist(new User("Bob", "bob@test.com", UserStatus.ACTIVE));
        tem.persist(new User("Alicia", "alicia@test.com", UserStatus.INACTIVE));
        tem.flush();

        List<UserView> results = repository.findByNameContaining("Ali");

        assertThat(results).hasSize(2);
        assertThat(results).extracting(UserView::getName)
                           .containsExactlyInAnyOrder("Alice", "Alicia");
    }

    @Test
    void shouldSupportSpecification() {
        tem.persist(new User("Charlie", "charlie@test.com", UserStatus.ACTIVE));
        tem.flush();

        var spec = UserViewSpecs.statusEq(UserStatus.ACTIVE)
                   .and(UserViewSpecs.nameLike("Char"));

        List<UserView> results = repository.findAll(spec);

        assertThat(results).hasSize(1);
        assertThat(results.get(0).getName()).isEqualTo("Charlie");
    }
}

陷阱 1：@Query 注解不可用

EntityViewRepository 不支持 Spring Data 的 @Query 注解。JPQL 字符串绕过了 Blaze-Persistence 的查询引擎，无法与 Entity View 投影协同。

// ❌ 不生效 — @Query 绕过 Entity View
@Query("SELECT u FROM User u WHERE u.name = :name")
List<UserView> findByNameCustom(@Param("name") String n);
// ✅ 使用方法名派生
List<UserView> findByName(String name);

// ✅ 或使用 Specification
List<UserView> findAll(
    EntityViewSpecification<UserView> spec);

陷阱 2：LazyInitializationException

LazyInitializationException 触发场景

解决方案：

  • 确保 Entity View 中所有属性都是 eager 加载的（Entity View 默认即 eager）
  • 避免在 View 中映射未声明的关联属性
  • 如果使用子视图，确保关联在查询时已经被 JOIN FETCH

陷阱 3：测试上下文缓存

// ❌ 每个测试类创建新的 CBF/EVM → 测试变慢
@Bean

    return Criteria.getDefault().createCriteriaBuilderFactory(emf);
}

// ✅ 正确做法：Singleton + @Lazy(false) 确保缓存复用
@Bean
@Scope(ConfigurableBeanFactory.SCOPE_SINGLETON)
@Lazy(false)
public CriteriaBuilderFactory cbf(EntityManagerFactory emf) {
    return Criteria.getDefault().createCriteriaBuilderFactory(emf);
}

本章小结

  • Spring Boot Starter 自动装配 CriteriaBuilderFactory、EntityViewManager 和 Repository 工厂，零配置即可启动
  • EntityViewRepository 替代 JpaRepository，所有查询自动投影为 Entity View，只 SELECT 需要的字段
  • 方法名查询派生通过 CriteriaBuilder 执行，继承 Blaze-Persistence 的 JOIN 优化能力
  • EntityViewSpecification 提供动态查询组合模式，适用于多条件筛选场景
  • Keyset 分页与 Spring Data Pageable 无缝桥接，O(1) 性能优于 OFFSET 分页
  • 事务管理：写操作必须在 @Transactional 中执行，只读查询使用 readOnly = true 获得性能提升
  • 测试配置：@DataJpaTest 需要额外导入 Blaze 配置类，CBF/EVM 应设为 Singleton 避免重复创建
  • 已知限制：@Query 注解不可用、注意版本兼容矩阵、避免延迟加载跨事务访问
