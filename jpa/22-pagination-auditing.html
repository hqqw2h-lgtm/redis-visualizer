第 22 章 · 分页、排序与审计

Offset · Slice · Keyset · @CreatedDate · Envers · 软删除

数据库的百万级订单不会一次全查出。分页和排序是生产系统的日常操作。审计(谁在何时创建、修改)是企业应用合规能力的基础。

本章从 Pageable 接口出发,剖析 Offset 与 Keyset 两种分页策略的性能差异,深入 Spring Data 审计机制和 Hibernate Envers 版本审计,最后补充软删除的三种实现方式。

  • 三种列表形态:Page(含 count) / Slice(size+1) / Keyset(WHERE 游标)
  • Offset 深分页 O(n) 退化的真实代价,以及 Keyset O(log n) 的实现细节
  • 排序白名单防注入 + 防全表扫描 + 性能可预测
  • Spring Data 审计:@CreatedDate / @LastModifiedDate / @CreatedBy / @LastModifiedBy
  • Hibernate Envers _AUD 表 + REVINFO 表的完整历史
  • 软删除三种实现(@SQLDelete + @Where / 字段标记 / 状态机)与唯一约束陷阱
  • OFFSET 深分页 vs Keyset 在 1000 万数据下的性能基准

前置知识:Spring Data Repository(第 20 章);SQL 生成与索引(第 18 章);事务(第 15 章)。

22.1 Pageable 与 Page

@RestController
public class OrderController {

    @Autowired
    private OrderRepository repo;

    @GetMapping("/orders")
    public Page<OrderDto> list(
        @PageableDefault(size = 20,
            sort = "createdAt",
            direction = Sort.Direction.DESC) Pageable pageable) {

        return repo.findByStatus(OrderStatus.PAID, pageable)
            .map(OrderDto::from);
    }
}
GET /orders?page=0&size=20&sort=createdAt,desc

HTTP/1.1 200
{
  "number": 0,
  "size": 20,
  "totalElements": 1284,
  "totalPages": 65,
  "content": [...]
}
参数含义默认
page页码(0-based)0
size每页大小20
sort排序字段unsorted
directionasc / descasc
 
-- content query
SELECT o.* FROM orders o
WHERE o.status = ?
ORDER BY o.created_at DESC
LIMIT 20 OFFSET 0;

-- count query
SELECT count(*) FROM orders o
WHERE o.status = ?;

Page = data + count 两条 SQL。Page 必须执行 count 才能算出 totalElements/totalPages,大表 count 可能慢到秒级。如果 UI 不需要总页数(信息流、加载更多),用 Slice 省掉 count。

22.2 Slice 与无限滚动

public interface OrderRepository extends JpaRepository<Order, Long> {

    Slice<Order> findByStatusOrderByCreatedAtDesc(
        OrderStatus status, Pageable pageable);
}

@RestController
public class OrderController {

    @GetMapping("/orders/slice")
    public Map<String, Object> slice(
        @PageableDefault(size = 20) Pageable pageable) {

        Slice<Order> slice = repo.findByStatusOrderByCreatedAtDesc(
            OrderStatus.PAID, pageable);

        return Map.of(
            "items", slice.getContent(),
            "hasNext", slice.hasNext(),
            "next", slice.hasNext() ? pageable.next() : null);
    }
}
维度PageSlice
count SQL是否
总页数提供不提供
hasNext通过 totalPagessize + 1 判断
性能(大表)慢快
UI 场景后台表格移动 / 信息流

22.4 排序白名单

public Page<Order> list(@PageableDefault(size = 20) Pageable pageable) {
    Pageable safe = sanitize(pageable);
    return repo.findAll(safe);
}

private Pageable sanitize(Pageable input) {
    Set<String> allowed = Set.of("id", "createdAt", "amount", "status");
    Sort filtered = Sort.by(input.getSort().stream()
        .filter(o -> allowed.contains(o.getProperty()))
        .toList());
    return PageRequest.of(input.getPageNumber(), input.getPageSize(), filtered);
}
目的说明
防 SQL 注入拒绝非白名单字段
防全表扫描拒绝无索引字段
性能可预测限定可优化字段
业务一致性拒绝业务无意义排序

22.5 Spring Data 审计

注解字段自动填充时机
@CreatedDateLocalDateTimepersist 时
@LastModifiedDateLocalDateTimepersist / update 时
@CreatedByString / Longpersist 时
@LastModifiedByString / Longpersist / update 时
 
@Configuration
@EnableJpaAuditing(auditorAwareRef = "auditorAware")
public class AuditConfig {

    @Bean
    public AuditorAware<String> auditorAware() {
        return () -> Optional.ofNullable(SecurityContextHolder.getContext())
            .map(SecurityContext::getAuthentication)
            .filter(Authentication::isAuthenticated)
            .map(Authentication::getName);
    }
}

@Entity
@EntityListeners(AuditingEntityListener.class)
public class Order {

    @Id @GeneratedValue
    private Long id;

    @CreatedDate
    @Column(updatable = false)
    private LocalDateTime createdAt;

    @LastModifiedDate
    private LocalDateTime updatedAt;

    @CreatedBy
    @Column(updatable = false, length = 50)
    private String createdBy;

    @LastModifiedBy
    @Column(length = 50)
    private String updatedBy;
}

线上现场:某 SaaS 系统迁移到 Spring Data 审计后发现 createdBy 全是 null。原因是没在 @SpringBootApplication 上加 @EnableJpaAuditing,导致 AuditingEntityListener 没注册。修复后立即生效。

@Audited
public class Order {

}
-- INSERT 时同步写
INSERT INTO orders ...;
INSERT INTO orders_aud (id, rev, revtype, amount, status) VALUES (..., 1, 0, ...);
INSERT INTO revinfo (rev, revtype, ...) VALUES (1, 0, ...);

-- revtype 含义
-- 0 = ADD, 1 = MOD, 2 = DEL
public List<Order> getOrderHistory(Long orderId) {
    AuditReader reader = AuditReaderFactory.get(em);
    List<Number> revisions = reader.getRevisions(Order.class, orderId);

    return revisions.stream()
        .map(rev -> reader.find(Order.class, orderId, rev))
        .toList();
}

22.7 软删除

策略实现适用
@SQLDelete + @WhereHibernate 注解简单
字段标记deleted_at 字段复杂业务
状态机status = DELETED状态多
 
@Entity
@Table(name = "orders")
@SQLDelete(sql = "UPDATE orders SET deleted_at = NOW() WHERE id = ?")
@Where(clause = "deleted_at IS NULL")
public class Order {

    @Id @GeneratedValue
    private Long id;

    private LocalDateTime deletedAt;
}
-- em.remove(order)
UPDATE orders SET deleted_at = NOW() WHERE id = ?;

-- em.find(Order.class, 1L)
SELECT o.* FROM orders o WHERE o.id = ? AND o.deleted_at IS NULL;
陷阱现象修复
唯一约束无法重新创建相同 key加 deleted_at 复合唯一
Native query不自动加 WHERE显式加条件
Cascade 不级联关联未删@SQLDelete on 关联
索引膨胀已删数据占空间物理清理任务

22.8 设计取舍

列表形态

后台表格 → Page;移动 / 信息流 → Slice;深分页 → Keyset。混合 UI 按主路径选,避免每个页面都跑 count。

审计粒度

Spring Data 审计仅记 4 个字段;Envers 记完整快照。Envers 数据量翻倍,需归档。小项目用 Spring Data 审计,合规严格用 Envers。

软删除是否物化

业务关键表(订单/用户)软删;日志/关联表物理删;定期清理软删数据归档冷存。

22.10 回归护栏

@Test
void pageShouldExecuteContentAndCountSql() {
    SqlRecorder recorder = sqlRecorder.clear();

    repo.findByStatus(OrderStatus.PAID, PageRequest.of(0, 20));

    long selectCount = recorder.countSqlContaining("select");
    assertThat(selectCount).isEqualTo(2L);
    assertThat(recorder.allSql()).anyMatch(s -> s.contains("limit"));
    assertThat(recorder.allSql()).anyMatch(s -> s.contains("count"));
}

@Test
void createdAtShouldBePopulatedOnPersist() {
    Order order = new Order(...);

    repo.saveAndFlush(order);

    assertThat(order.getCreatedAt()).isNotNull();
    assertThat(order.getCreatedBy()).isEqualTo("test-user");
}

@Test
void updatedAtShouldChangeOnUpdate() throws InterruptedException {
    Order order = repo.findById(1L).orElseThrow();
    LocalDateTime original = order.getUpdatedAt();

    Thread.sleep(100);
    order.setStatus("PAID");
    repo.saveAndFlush(order);

    assertThat(order.getUpdatedAt()).isAfter(original);
}
