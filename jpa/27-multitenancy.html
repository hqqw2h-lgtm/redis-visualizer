第 27 章 · 多租户与数据隔离

Discriminator · Schema · Database · TenantContext · 缓存隔离

SaaS 多租户最怕的不是查不到数据,而是查到了不该看的数据。JPA / Hibernate 能将租户标识传入 EntityManager、连接、SQL、缓存边界,但可靠的设计必须把解析、路由、SQL 谓词、回归测试串成闭环。

  • 三种隔离策略对比:Discriminator(共享表) / Schema(共享 DB) / Database(独立实例)
  • 租户标识请求级不变量:Filter → ThreadLocal → CurrentTenantIdentifierResolver
  • Hibernate 6 @TenantId 注解的自动 INSERT/SELECT/UPDATE/DELETE 谓词
  • MultiTenantConnectionProvider 切换 PostgreSQL search_path
  • RoutingDataSource 实现 Database 级隔离
  • L2 缓存 / Query Cache / NaturalId Cache 自动加 tenantId 隔离
  • 异步任务的 TenantContext 传递(Executor 装饰器)
  • 多租户 Flyway 迁移(按 schema 循环)
  • 跨租户报表与租户上下文清理护栏

前置知识:HikariCP(第 19 章);Hibernate 启动链(第 17 章);ThreadLocal 编程。

27.1 租户标识请求级不变量

public final class TenantContext {

    private static final ThreadLocal<String> CURRENT = new ThreadLocal<>();

    private TenantContext() {}

    public static void set(String tenantId) {
        Objects.requireNonNull(tenantId);
        CURRENT.set(tenantId);
    }

    public static String require() {
        String t = CURRENT.get();
        if (t == null) throw new IllegalStateException("Tenant context missing");
        return t;
    }

    public static void clear() { CURRENT.remove(); }
}

@Component
@Order(Ordered.HIGHEST_PRECEDENCE)
public class TenantFilter extends OncePerRequestFilter {

    private static final String HEADER = "X-Tenant-Id";

    @Override
    protected void doFilterInternal(HttpServletRequest req, HttpServletResponse resp,
                                     FilterChain chain) throws ServletException, IOException {

        String tenantId = req.getHeader(HEADER);
        if (tenantId == null || tenantId.isBlank()) {
            resp.sendError(SC_BAD_REQUEST, "missing X-Tenant-Id");
            return;
        }

        try {
            TenantContext.set(tenantId);
            chain.doFilter(req, resp);
        } finally {
            TenantContext.clear();
        }
    }
}

27.2 Discriminator 策略 — Hibernate 6 @TenantId

@Entity
@Table(name = "orders")
public class Order {
    @Id @GeneratedValue
    private Long id;

    @TenantId
    @Column(nullable = false, updatable = false)
    private String tenantId;

    private BigDecimal amount;
}

@Component
public class CustomTenantResolver implements CurrentTenantIdentifierResolver<String> {
    @Override
    public String resolveCurrentTenantIdentifier() { return TenantContext.require(); }

    @Override
    public boolean validateExistingCurrentSessions() { return true; }
}
spring:
  jpa:
    properties:
      hibernate:
        tenant_identifier_resolver: com.acme.CustomTenantResolver
        multi_tenancy: DISCRIMINATOR
-- INSERT 自动加 tenant_id
INSERT INTO orders (id, tenant_id, amount) VALUES (?, 'tenant-cn', ?);

-- SELECT/UPDATE/DELETE 自动加 WHERE
SELECT o.* FROM orders o WHERE o.id = ? AND o.tenant_id = 'tenant-cn';
UPDATE orders SET amount = ? WHERE id = ? AND tenant_id = 'tenant-cn';

Discriminator 的弱点:native query 不自动加 WHERE,需手写过滤。任何漏写都会跨租户泄漏 → ① 全员 review ② Filter 兜底 ③ 定期渗透测试。

27.3 Schema 策略 — search_path 切换

图 27-3-1 · 借连接 → SET search_path → 查询 → 归还前重置
@Component
public class SchemaTenantConnectionProvider
    implements MultiTenantConnectionProvider<String> {

    @Autowired private DataSource dataSource;

    @Override
    public Connection getConnection(String tenantId) throws SQLException {
        Connection conn = dataSource.getConnection();
        try (Statement st = conn.createStatement()) {
            st.execute("SET search_path TO " + sanitize(tenantId));
        }
        return conn;
    }

    @Override
    public void releaseConnection(String tenantId, Connection conn) throws SQLException {
        try (Statement st = conn.createStatement()) {
            st.execute("SET search_path TO public");
        }
        conn.close();
    }

    private String sanitize(String tenantId) {
        if (!tenantId.matches("[a-z0-9_]+"))
            throw new IllegalArgumentException("invalid tenantId");
        return tenantId;
    }
}

27.4 Database 策略 — RoutingDataSource

public class TenantRoutingDataSource extends AbstractRoutingDataSource {
    @Override
    protected Object determineCurrentLookupKey() { return TenantContext.require(); }
}

@Configuration
public class DataSourceConfig {

    @Bean
    public DataSource routingDataSource() {
        Map<Object, Object> targets = new HashMap<>();
        targets.put("tenant-cn", buildPool("jdbc:postgresql://cn-db/orders"));
        targets.put("tenant-eu", buildPool("jdbc:postgresql://eu-db/orders"));
        targets.put("tenant-us", buildPool("jdbc:postgresql://us-db/orders"));

        TenantRoutingDataSource ds = new TenantRoutingDataSource();
        ds.setTargetDataSources(targets);
        ds.setDefaultTargetDataSource(targets.get("tenant-cn"));
        return ds;
    }

    private DataSource buildPool(String url) {
        HikariConfig cfg = new HikariConfig();
        cfg.setJdbcUrl(url);
        cfg.setMaximumPoolSize(10);
        return new HikariDataSource(cfg);
    }
}
项注意
DataSource 数量每租户一个

27.6 设计取舍

Discriminator(共享表)

优势:成本低、跨租户报表方便、单库维护。代价:native query 漏过滤即泄漏、性能受邻租户影响、单表过大。

Schema(共享 DB,独立 schema)

优势:中等隔离、Flyway 按 schema 循环迁移。代价:运维稍复杂、schema 数量有上限(PG ~1000)。

Database(独立实例)

优势:强隔离、故障互不影响、合规友好。代价:运维成本 N 倍、连接池总数膨胀、跨租户报表需 ETL。

27.8 回归护栏

@Test
void differentTenantsShouldNotSeeEachOthersOrders() {
    TenantContext.set("tenant-A");
    Long idA = orderRepo.save(new Order(BigDecimal.TEN)).getId();
    TenantContext.clear();

    TenantContext.set("tenant-B");
    Optional<Order> result = orderRepo.findById(idA);
    TenantContext.clear();

    assertThat(result).isEmpty();
}

@Test
void selectShouldIncludeTenantPredicate() {
    TenantContext.set("tenant-A");
    SqlRecorder recorder = sqlRecorder.clear();
    orderRepo.findById(1L);
    assertThat(recorder.lastSql().toLowerCase()).contains("tenant_id");
    TenantContext.clear();
}

27.9 异步任务上下文传递

@Component
public class TenantAwareTaskExecutor extends ThreadPoolTaskExecutor {

    @Override
    public void execute(Runnable task) {
        String tenantId = TenantContext.require();
        super.execute(() -> {
            try {
                TenantContext.set(tenantId);
                task.run();
            } finally {
                TenantContext.clear();
            }
        });
    }
}
// 跨租户报表
@Service
public class CrossTenantReportService {

    public Map<String, Long> countOrdersByTenant() {
        Map<String, Long> result = new HashMap<>();
        for (String tenant : listTenants()) {
            try {
                TenantContext.set(tenant);
                result.put(tenant, orderRepo.count());
            } finally {
                TenantContext.clear();
            }
        }
        return result;
    }
}
