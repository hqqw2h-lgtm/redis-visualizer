第 17 章 · Hibernate 架构全景

SessionFactory 单例 · Session 工作单元 · ActionQueue · EventListener · SQM + SQL AST · Statistics

Hibernate 是 JPA 的参考实现。本章系统拆解 SessionFactory 构建链、SessionImpl 工作单元、事件监听系统、Action Queue 写入队列、查询引擎、SPI 扩展点。

排查路径是把问题落到组件层:启动慢看 Metadata,内存涨看 PersistenceContext,SQL 异常看 QueryEngine,顺序异常看 ActionQueue,审计扩展看事件系统。

  • SessionFactory 构建链:StandardServiceRegistry → MetadataSources → Metadata → SessionFactoryImpl
  • SessionImpl 内部组件:PersistenceContext + ActionQueue + EventSource + TransactionCoordinator + JdbcCoordinator
  • 8 类核心事件:PreInsert/Update/Delete/Load + Flush + AutoFlush + DirtyCheck + Persist
  • ActionQueue 的 6 阶段严格顺序:Insert → Update → CollDel → CollUpd → CollIns → Delete
  • Hibernate 6 的 SQM(语义查询模型)+ SQL AST 两阶段编译模型
  • 8 大 SPI 扩展点:Interceptor / EventListener / StatementInspector / Dialect / IdGen / Converter / MultiTenant
  • Hibernate Statistics 8 项指标 + Micrometer 暴露
  • Hibernate 3.x → 6.4 的演进时间线 + 三大设计取舍
  • SessionFactory 重 Session 轻、Action 延迟写入、事件系统而非 AOP 的源头
  • 启动配置推荐 9 项 + Bytecode Enhancement 配置

前置知识:第 3 章 PersistenceContext;第 14 章批量与 JDBC Batching;Spring AOP 与 IoC 容器基础;ANTLR 解析器与 AST 的基本概念。

public final class AuditPreInsertListener implements PreInsertEventListener {

    @Override
    public boolean onPreInsert(PreInsertEvent event) {
        if (event.getEntity() instanceof Auditable auditable) {
            auditable.setCreatedBy(SecurityContext.currentUser());
            auditable.setCreatedAt(Instant.now());
        }
        return false;
    }
}

@Configuration
public class HibernateEventConfig {

    @Autowired
    private EntityManagerFactory emf;

    @PostConstruct
    public void register() {
        SessionFactoryImpl sfi = emf.unwrap(SessionFactoryImpl.class);
        EventListenerRegistry registry = sfi.getServiceRegistry()
            .getService(EventListenerRegistry.class);
        registry.appendListeners(EventType.PRE_INSERT,
            new AuditPreInsertListener());
    }
}

@Bean
MeterBinder hibernateMetrics(EntityManagerFactory emf) {
    return registry -> {
        SessionFactory sf = emf.unwrap(SessionFactory.class);
        Statistics stats = sf.getStatistics();
        Gauge.builder("hibernate.entity.inserts", stats,
                Statistics::getEntityInsertCount)
            .register(registry);

                Statistics::getQueryCacheHitCount)
            .register(registry);
    };
}

17.9 设计取舍

取舍 1:SessionFactory 重 / Session 轻。SessionFactory 是线程安全的应用级单例,保留映射元数据 / 缓存 / 连接池;Session 是非线程安全的请求级对象,保留 PersistenceContext 与事务边界。这种二元设计借鉴了 JDBC ConnectionPool / Connection 模型,平衡资源开销与并发模型。

取舍 2:Action 队列与延迟写入。不立即发 SQL 而是排队,目的是 ① 启用 JDBC Batching ② 自动重排避免 FK 冲突 ③ 与 dirty checking 配合实现透明持久化。代价是无法在业务代码中精确控制 SQL 时机,需要 flush() / flushMode 显式干预。

取舍 3:事件系统而非 AOP。Hibernate 选择事件系统而非 Spring AOP 实现钩子。原因:事件触发点比方法调用更精细(例如 dirty checking 后才能触发 PreUpdate),且不依赖代理。代价是配置复杂,需要 EventListenerRegistry 编程注册。
