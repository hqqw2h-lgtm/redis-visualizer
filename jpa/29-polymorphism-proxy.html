第 29 章 · 多态查询与代理陷阱

SINGLE_TABLE / JOINED / TPC · HibernateProxy · 四颗地雷 · DTO 边界

JPA 的继承映射让 Java 多态优雅,但代理机制埋下四颗地雷:instanceof 失效、equals 失效、类型窄化、JSON 序列化丢字段。每一个都可能让生产系统在深夜报警。

本章是全书最重要的避坑专题。先理解多态查询在三种继承策略下的 SQL 生成与性能差异,再逐一拆解 HibernateProxy 的四大陷阱及解决方案。

  • SINGLE_TABLE / JOINED / TABLE_PER_CLASS 三策略 SQL 生成与性能基准
  • HibernateProxy 的类型继承层次:为什么 Proxy 仅继承声明类型
  • 陷阱 1:em.getReference(Payment, id) instanceof CreditCardPayment 永远 false
  • 陷阱 2:equals 用业务字段触发 lazy load,Set/HashMap 行为不稳
  • 陷阱 3:类型窄化 — PersistenceContext 中已有 Payment 代理,find(子类) 仍返父类型
  • 陷阱 4:JSON 序列化代理对象出现 hibernateLazyInitializer + LazyInitializationException
  • TYPE() 多态过滤、Visitor 模式替代 instanceof、Hibernate 6 字节码增强
  • 三道护栏:Hibernate.unproxy + 仅按 ID equals + DTO 边界

前置知识:JPA 继承映射(@Inheritance);Hibernate 代理与字节码增强;Java equals/hashCode 契约。

29.1 三策略多态 SQL

@Entity
@Inheritance(strategy = InheritanceType.SINGLE_TABLE)
@DiscriminatorColumn(name = "dtype")
abstract class Payment {
    @Id @GeneratedValue private Long id;
    private BigDecimal amount;
}

@DiscriminatorValue("CREDIT")
@Entity
class CreditCardPayment extends Payment {
    private String cardNumber;
}
-- SINGLE_TABLE
SELECT p.id, p.dtype, p.amount, p.card_number, p.bank_account, ...
FROM payments p;

-- JOINED
SELECT p.id, p.amount,
       cp.card_number, bp.bank_account, wp.wallet_no,
       CASE
         WHEN cp.id IS NOT NULL THEN 'CreditCard'
         WHEN bp.id IS NOT NULL THEN 'BankTransfer'
         WHEN wp.id IS NOT NULL THEN 'Wallet'
       END AS clazz
FROM payments p
LEFT JOIN credit_card_payments cp ON p.id = cp.id
LEFT JOIN bank_transfer_payments bp ON p.id = bp.id
LEFT JOIN wallet_payments wp ON p.id = wp.id;

-- TABLE_PER_CLASS
SELECT * FROM (
    SELECT id, amount, card_number, NULL::varchar AS bank_account, NULL AS wallet_no, 'CreditCard' AS clazz
        FROM credit_card_payments
    UNION ALL
    SELECT id, amount, NULL, bank_account, NULL, 'BankTransfer'
        FROM bank_transfer_payments
    UNION ALL
    SELECT id, amount, NULL, NULL, wallet_no, 'Wallet'
        FROM wallet_payments
) p ORDER BY p.amount DESC;
策略SQL性能NULL 列索引可用
SINGLE_TABLE单表最快多优
JOINED多 LEFT JOIN中少中
TABLE_PER_CLASSUNION ALL慢NULL 占位差

29.2 陷阱 1 — instanceof 失效

Payment payment = em.getReference(Payment.class, 1L);

// payment 是 Payment$HibernateProxy(继承 Payment,不继承 CreditCardPayment)
if (payment instanceof CreditCardPayment) {     // ❌ 返回 false
    CreditCardPayment cp = (CreditCardPayment) payment;
}

// ✅ 解包后再判断
Payment unproxy = Hibernate.unproxy(payment, Payment.class);
if (unproxy instanceof CreditCardPayment cp) {
    String cardNumber = cp.getCardNumber();
}

为什么 Proxy 不继承子类?因为创建代理时只知道声明类型 Payment.class,Hibernate 不能预知数据库中实际是哪个子类。代理通过 id 查父表 dtype,加载后才知真实类型,但 Java 类型已固定。

29.3 陷阱 2 — equals / hashCode

// ❌ 错误:用业务字段
@Override
public boolean equals(Object o) {
    if (o instanceof Order other) {
        return Objects.equals(id, other.id)
            && Objects.equals(orderNo, other.orderNo);   // 触发 lazy load
    }
    return false;
}

// ✅ 正确:仅按 @Id + Hibernate.getClass
@Override
public boolean equals(Object o) {
    if (this == o) return true;
    if (o == null) return false;

    Class<?> thisClass = Hibernate.getClass(this);
    Class<?> otherClass = Hibernate.getClass(o);
    if (!thisClass.equals(otherClass)) return false;

    Order other = (Order) o;
    return id != null && id.equals(other.id);
}

@Override
public int hashCode() {
    // 不能用 id(可能 null + 持久化后改变)
    return getClass().hashCode();
}
规则原因
仅按 @Id 比较代理访问 ID 不触发 lazy load
用 Hibernate.getClass解包代理类
hashCode 用类 hashCode不依赖 ID(可能 null)
不用业务字段触发 lazy load
父类 / 子类同 hashCode一致性

29.4 陷阱 3 — 类型窄化

图 29-4-1 · getReference → PC 已存 Proxy → find 子类型仍返代理
// ❌ 类型窄化失败
Payment lazy = em.getReference(Payment.class, 1L);
CreditCardPayment found = em.find(CreditCardPayment.class, 1L);
assertThat(found instanceof CreditCardPayment).isTrue();   // false
assertThat(found).isSameAs(lazy);                          // true(同一 Payment Proxy)

// ✅ 修复:clear 后重新 find
em.clear();
CreditCardPayment found = em.find(CreditCardPayment.class, 1L);

29.5 陷阱 4 — JSON 序列化

// ❌ 直接返回实体
@GetMapping("/orders/{id}")
public Order get(@PathVariable Long id) {
    return repo.findById(id).orElseThrow();
}

// 序列化后的 JSON 出现:
// "hibernateLazyInitializer": { ... },
// "handler": null
// 同时事务外触发 LazyInitializationException

// ✅ DTO 边界
public record OrderDto(
    Long id,
    BigDecimal amount,
    String customerName,
    List<OrderItemDto> items
) {
    public static OrderDto from(Order o) {
        return new OrderDto(
            o.getId(),
            o.getAmount(),
            o.getCustomer().getName(),
            o.getItems().stream().map(OrderItemDto::from).toList());
    }
}

@GetMapping("/orders/{id}")
public OrderDto get(@PathVariable Long id) {
    Order order = repo.findDetailById(id).orElseThrow();
    return OrderDto.from(order);
}
序列化方案优势劣势
DTO 边界安全 / 显式代码量
Jackson HibernateModule自动处理代理隐式
@JsonIgnore灵活不易维护
Open Projection (SpEL)简洁性能差

29.6 TYPE() 多态过滤

// 仅查特定子类
em.createQuery("select p from Payment p where TYPE(p) = CreditCardPayment", Payment.class);

// 排除某子类
em.createQuery("select p from Payment p where TYPE(p) <> CreditCardPayment", Payment.class);

// 多类型 IN
em.createQuery("""
    select p from Payment p
    where TYPE(p) IN (CreditCardPayment, WalletPayment)
    """, Payment.class);
-- SINGLE_TABLE 渲染
SELECT * FROM payments WHERE dtype = 'CREDIT';

-- JOINED 渲染(更贵)
SELECT p.*, cp.* FROM payments p
JOIN credit_card_payments cp ON p.id = cp.id;

29.8 Visitor 模式替代 instanceof

public interface Payment {
    <R> R accept(PaymentVisitor<R> visitor);
}

public interface PaymentVisitor<R> {
    R visit(CreditCardPayment p);
    R visit(BankTransferPayment p);
    R visit(WalletPayment p);
}

@Component
public class FraudCheckVisitor implements PaymentVisitor<RiskLevel> {

    @Override
    public RiskLevel visit(CreditCardPayment p) { return cardRiskService.check(p.getCardNumber()); }

    @Override
    public RiskLevel visit(BankTransferPayment p) { return bankRiskService.check(p.getBankAccount()); }

    @Override
    public RiskLevel visit(WalletPayment p) { return walletRiskService.check(p.getWalletNo()); }
}

// 安全 instanceof 工具
public static <T> Optional<T> safeCast(Object obj, Class<T> targetType) {
    Object real = Hibernate.unproxy(obj);
    if (targetType.isInstance(real)) return Optional.of(targetType.cast(real));
    return Optional.empty();
}
