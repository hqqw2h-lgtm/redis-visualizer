第 28 章 · Spring Data JPA 高级特性

@EntityGraph · Dynamic Projection · SpEL · Custom Fragment · Lifecycle · Stream

Spring Data JPA 的高级能力不是少写几行代码,而是把查询入口、fetch plan、返回形状、回归测试收束到稳定边界。上线时盯住三件事:生成了什么 SQL、返回了什么 shape、一次请求打了几条 SQL。

  • 5 种入口对比:派生查询 / @Query JPQL / @Query Native / Specification / Custom Fragment
  • @EntityGraph LOAD vs FETCH 的语义差异,以及 NamedEntityGraph vs ad-hoc attributePaths
  • 动态 Projection:<T> Class<T> type 参数让一个方法支持多种返回形态
  • 嵌套 interface projection 只 SELECT 必要列
  • SpEL 参数 :#{principal.tenantId} 自动注入当前用户上下文
  • @NoRepositoryBean BaseRepository + #{#entityName} 跨实体通用查询
  • Custom Fragment 命名约定(*Impl 后缀)与 EntityManager 注入
  • 7 个 Lifecycle 注解(@PrePersist/@PostLoad/@PreUpdate...)的触发时机
  • Stream 处理百万行结果(HINT_FETCH_SIZE + try-with-resources)

前置知识:Spring Data Repository(第 20 章);Specification(第 21 章);Criteria API。

28.5 Custom Fragment

public interface OrderRepositoryCustom {
    List<Order> searchByCriteria(OrderSearchForm form, Pageable pageable);
    int bulkUpdateStatus(List<Long> ids, OrderStatus newStatus);
}

// 命名约定: *Impl 后缀
@Component
public class OrderRepositoryCustomImpl implements OrderRepositoryCustom {

    @PersistenceContext
    private EntityManager em;

    @Override
    public List<Order> searchByCriteria(OrderSearchForm form, Pageable pageable) {
        CriteriaBuilder cb = em.getCriteriaBuilder();
        CriteriaQuery<Order> cq = cb.createQuery(Order.class);
        Root<Order> root = cq.from(Order.class);
        List<Predicate> ps = new ArrayList<>();

        if (form.getStatus() != null) {
            ps.add(cb.equal(root.get(Order_.status), form.getStatus()));
        }
        cq.where(cb.and(ps.toArray(Predicate[]::new)))
          .orderBy(cb.desc(root.get(Order_.createdAt)));

        return em.createQuery(cq)
            .setFirstResult((int) pageable.getOffset())
            .setMaxResults(pageable.getPageSize())
            .getResultList();
    }

    @Override
    public int bulkUpdateStatus(List<Long> ids, OrderStatus newStatus) {
        return em.createQuery("""
            update Order o set o.status = :status
            where o.id in :ids
            """)
            .setParameter("status", newStatus)
            .setParameter("ids", ids)
            .executeUpdate();
    }
}

// Repository 接口同时继承基础与 Custom
public interface OrderRepository
    extends JpaRepository<Order, Long>, OrderRepositoryCustom {
}

28.6 Lifecycle 事件

注解触发用途
@PrePersistpersist 前默认值 / 校验
@PostPersistpersist 后通知 / 索引
@PreUpdateflush 前(脏检查后)审计 / 校验
@PostUpdateUPDATE 执行后通知
@PreRemoveremove 前清理
@PostRemoveDELETE 后清理
@PostLoadfind / query 后数据补全
 
@Entity
public class Order {

    @PrePersist
    void prePersist() {
        if (orderNo == null) orderNo = generateOrderNo();
        if (createdAt == null) createdAt = LocalDateTime.now();
    }

    @PostLoad
    void postLoad() {
        // 加载后初始化派生字段
        this.displayLabel = computeLabel();
    }
}

禁忌:不要在 Lifecycle 钩子中执行 JPA 查询/更新。当前正在 flush,再发 SQL 会触发 ConcurrentModificationException 或死锁。Lifecycle 仅用于修改自身字段。

28.7 Stream 大结果集

public interface OrderRepository extends JpaRepository<Order, Long> {

    @QueryHints(@QueryHint(name = HINT_FETCH_SIZE, value = "100"))
    Stream<Order> findAllByStatus(OrderStatus status);
}

@Transactional(readOnly = true)
public void processAll(OrderStatus status) {
    try (Stream<Order> stream = repo.findAllByStatus(status)) {
        stream.forEach(o -> processSingle(o));
    }
}

Stream 处理百万行:① HINT_FETCH_SIZE 控制每次 JDBC fetch 多少行;② try-with-resources 保证 ResultSet 与 PersistenceContext 关闭;③ 处理完后用 em.clear() 防止 PC 内存溢出。

28.8 通用 BaseRepository

@NoRepositoryBean
public interface BaseRepository<T, ID> extends JpaRepository<T, ID> {

    @Query("select e from #{#entityName} e where e.deleted = false")
    List<T> findAllNotDeleted();

    @Query("select count(e) from #{#entityName} e where e.deleted = false")
    long countNotDeleted();

    default Optional<T> findActiveById(ID id) {
        return findById(id).filter(e -> !((SoftDeletable) e).isDeleted());
    }
}

// 子接口自动获得通用方法
public interface OrderRepository extends BaseRepository<Order, Long> {}
public interface CustomerRepository extends BaseRepository<Customer, Long> {}
