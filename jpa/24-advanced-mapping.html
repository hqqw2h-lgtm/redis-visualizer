第 24 章 · Hibernate 高级映射

业务键 · 派生列 · 动态过滤 · 多表 · 软删除 · 多态 · 动态 DML

标准 JPA 注解覆盖 80% 的映射需求,剩下 20% 的真实场景需要 Hibernate 扩展注解:业务主键查找、计算属性、动态过滤、多表映射、软删除、多态关联。

这些注解不是可选的高级玩具,而是生产系统中日复一日被使用的工具。本章逐一拆解七个核心扩展,配以流程图、表结构、SQL 验证。

  • @NaturalId + @NaturalIdCache 让 byNaturalId 查询命中二级缓存
  • @Formula 的 SELECT 表达式如何插入 SQL,以及它带来的性能代价
  • @Filter(参数化运行时) vs @SQLRestriction(静态默认)的本质差异
  • @SecondaryTable 拆大表 → 自动 LEFT JOIN 多表的代价
  • @SoftDelete(Hibernate 6.4+ 内置)vs @SQLDelete + @Where 老方案
  • @Any / @ManyToAny 无继承多态关联与查询代价
  • @DynamicUpdate 只更新变化字段,但会导致 PreparedStatement 缓存膨胀
  • @TenantId(Hibernate 6 多租户)与 @Filter 的取舍
  • @ColumnTransformer 读写 SQL 转换(加密列经典场景)

前置知识:JPA 标准注解(第 4 章)、二级缓存(第 13 章)、SQL 生成(第 18 章)。

24.1 @NaturalId — 业务键查询 + 缓存

图 24-1-1 · byNaturalId 缓存命中 — sku → id → entity,无 SQL
@Entity
@NaturalIdCache
@Cache(usage = CacheConcurrencyStrategy.READ_WRITE)
public class Product {

    @Id @GeneratedValue
    private Long id;

    @NaturalId
    @Column(nullable = false, unique = true)
    private String sku;

    private String name;
}

Product p = session.byNaturalId(Product.class)
    .using("sku", "SKU-2026-001")
    .load();

24.2 @Formula — SELECT 表达式派生字段

@Entity
@Table(name = "orders")
public class Order {

    @Id @GeneratedValue
    private Long id;

    private BigDecimal subtotal;
    private BigDecimal taxRate;

    @Formula("subtotal + (subtotal * tax_rate / 100)")
    private BigDecimal totalAmount;

    @Formula("(SELECT count(*) FROM order_items i WHERE i.order_id = id)")
    private Integer itemCount;
}
SELECT o.id, o.subtotal, o.tax_rate,
       o.subtotal + (o.subtotal * o.tax_rate / 100) AS totalAmount,
       (SELECT count(*) FROM order_items i WHERE i.order_id = o.id) AS itemCount
FROM orders o
WHERE o.id = ?;

子查询 @Formula 慢。每次 SELECT 实体都跑一次子查询,对 100 行结果集就是 100 个子查询。如果是热路径,考虑物化为冗余字段 + 触发器维护,或改 @Subselect。

24.3 @Filter — 动态参数化过滤

@Entity
@Table(name = "orders")
@FilterDef(name = "tenantFilter",
    parameters = @ParamDef(name = "tenantId", type = Long.class))
@Filter(name = "tenantFilter", condition = "tenant_id = :tenantId")
public class Order {
    @Id @GeneratedValue private Long id;
    @Column(name = "tenant_id", nullable = false) private Long tenantId;
}

@Service
public class TenantContext {
    @Autowired private EntityManager em;

    public void setCurrentTenant(Long tenantId) {
        Session session = em.unwrap(Session.class);
        Filter filter = session.enableFilter("tenantFilter");
        filter.setParameter("tenantId", tenantId);
    }
}
-- 启用 Filter 后所有 Order 查询自动附加
SELECT o.* FROM orders o WHERE o.id = ? AND o.tenant_id = ?;
SELECT o.* FROM orders o WHERE o.status = ? AND o.tenant_id = ?;
维度@Filter@SQLRestriction
启用enableFilter 显式默认应用
参数化是否
适用多租户 / 权限软删除 / 状态过滤
关闭disableFilter不可关

24.4 @SecondaryTable — 一实体多表

@Entity
@Table(name = "products")
@SecondaryTable(name = "product_details",
    pkJoinColumns = @PrimaryKeyJoinColumn(name = "product_id"))
@SecondaryTable(name = "product_seo",
    pkJoinColumns = @PrimaryKeyJoinColumn(name = "product_id"))
public class Product {

    @Id @GeneratedValue
    private Long id;

    @Column(table = "products")
    private String sku;

    @Column(table = "product_details")
    private String description;

    @Column(table = "product_seo")
    private String metaTitle;
}
-- INSERT 三表
INSERT INTO products (id, sku) VALUES (1, 'X');
INSERT INTO product_details (product_id, description) VALUES (1, '...');
INSERT INTO product_seo (product_id, meta_title) VALUES (1, '...');

-- SELECT 自动 LEFT JOIN
SELECT p.id, p.sku, d.description, s.meta_title
FROM products p
LEFT JOIN product_details d ON p.id = d.product_id
LEFT JOIN product_seo s ON p.id = s.product_id
WHERE p.id = ?;

24.5 @SoftDelete — Hibernate 6.4+ 内置软删除

@Entity
@Table(name = "orders")
@SoftDelete(columnName = "deleted")
public class Order {
    @Id @GeneratedValue private Long id;
    private BigDecimal amount;
}
-- em.remove(order)
UPDATE orders SET deleted = true WHERE id = ?;

-- em.find / 任意查询自动加条件
SELECT o.* FROM orders o WHERE o.id = ? AND o.deleted = false;
方案实现优势
@SQLDelete + @Where手写 SQLHibernate 5+ 兼容
@SoftDelete注解Hibernate 6.4+,语义清晰
字段标记业务代码完全可控

24.6 @Any — 无继承多态关联

@Entity
@Table(name = "tasks")
public class Task {

    @Id @GeneratedValue
    private Long id;

    @Any
    @AnyDiscriminator(DiscriminatorType.STRING)
    @AnyDiscriminatorValue(discriminator = "ORDER", entity = Order.class)
    @AnyDiscriminatorValue(discriminator = "INVOICE", entity = Invoice.class)
    @JoinColumn(name = "target_id")
    @Column(name = "target_type")
    private Object target;
}
-- INSERT
INSERT INTO tasks (id, target_type, target_id) VALUES (?, 'ORDER', 100);

-- SELECT 两步:先查 task,再按 type 加载关联实体
SELECT id, target_type, target_id FROM tasks WHERE id = ?;
-- 然后按 type 加载 Order/Invoice

24.8 @Generated — 数据库生成字段

@Entity
public class AuditEntity {

    @Id @GeneratedValue
    private Long id;

    @Generated(GenerationTime.INSERT)
    @Column(insertable = false,
            columnDefinition = "TIMESTAMP DEFAULT CURRENT_TIMESTAMP")
    private LocalDateTime createdAt;

    @Generated(GenerationTime.ALWAYS)
    @Column(insertable = false, updatable = false,
            columnDefinition = "TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP")
    private LocalDateTime updatedAt;
}

24.9 @ColumnTransformer — 读写 SQL 转换(加密列)

@Entity
public class User {
    @Id @GeneratedValue private Long id;

    @ColumnTransformer(
        read  = "pgp_sym_decrypt(email::bytea, current_setting('app.encrypt_key'))",
        write = "pgp_sym_encrypt(?, current_setting('app.encrypt_key'))"
    )
    private String email;
}
INSERT INTO users (id, email)
VALUES (?, pgp_sym_encrypt(?, current_setting('app.encrypt_key')));

SELECT id, pgp_sym_decrypt(email::bytea, current_setting('app.encrypt_key'))
FROM users WHERE id = ?;

24.10 陷阱与限制

#陷阱现象修复
1NaturalId 缓存未启用总查 DB加 @NaturalIdCache
2@Formula 影响性能子查询慢加索引或物化
3@Filter 忘启用数据泄漏拦截器统一启用
4@SecondaryTable LEFT JOIN总查多表仅必要时用
5@SoftDelete native query不过滤手动加条件
6@Any 性能差无 join 优化按业务批量加载
7@DynamicUpdate 缓存膨胀PS 缓存大字段少时不开
8@Generated 不读回字段为 nullinsertable=false

24.11 本章要点

要点注解易错点
@NaturalId 业务键+ @NaturalIdCache缓存未启用
@Formula 派生字段SQL 表达式子查询慢
@Filter 动态过滤enableFilter忘启用 → 数据泄漏
@SecondaryTable 多表pkJoinColumns总 JOIN 多表
@SoftDelete 声明Hibernate 6.4+native query 不过滤
@Any 无继承多态AnyDiscriminator查询慢
@DynamicUpdate仅 dirty 字段PS 缓存膨胀
@TenantId(6+)自动多租户updatable=false
