第 20 章 · Spring Data JPA 核心

Repository 抽象 · 代理生成 · 方法名 DSL · @Query · Projection · Pageable · Custom Fragment

Spring Data JPA 在 JPA 规范之上构建了一层 Repository 抽象。开发者声明接口,Spring 启动时自动生成代理实现,大幅减少模板代码。

本章从接口层次、代理生成、方法名查询、@Query、Projection 投影到自定义 Fragment 全链路展开,并在每条路径上给出 SQL 行为与吞吐量化。验收标准是:看到接口方法,能预测分派路径、SQL 数量、是否额外 SELECT、P95 延迟数量级。

  • 5 层 Repository 接口层次:Repository → CrudRepository → ListCrud → PagingAndSorting → JpaRepository
  • JpaRepositoryFactoryBean 启动代理生成 + JDK 动态代理 + QueryLookup 路径分派
  • 方法名 DSL(PartTree)解析:findBy / Top / OrderBy / Containing / IgnoreCase 等关键字
  • @Query 三种模式:JPQL / Native SQL / @NamedQuery,以及 SpEL 支持
  • 3 种 Projection:DTO 构造投影 / Interface-based 接口投影 / 动态 ResultClass
  • Pageable 分页的 2 条 SQL(主查 + count)与 Sort 推断
  • Custom Fragment + 自定义实现类的命名约定(RepoCustomImpl)
  • 派生查询 vs @Query vs Specification 三者的取舍
  • 常见陷阱:派生查询参数过多、@Query 缺 countQuery、Projection 返回懒加载
  • Spring Data JPA 与 Hibernate 的协作边界:Spring 管 Repository 代理,Hibernate 管 SQL 生成

前置知识:JPA EntityManager API;Spring AOP / 动态代理;JPQL 基础;第 17 章 Hibernate 架构;第 18 章 SQL 生成。

派生查询的边界:超过 3 个 AND 条件、含 join、含子查询的方法名应改用 @Query。一句 findByCustomerStatusAndOrderStatusAndCreatedAtBetweenAndRegionInOrderByCreatedAtDesc 无法保证可读性,且属性变更时编译期无法发现。

20.3 @Query 注解

public interface OrderRepository extends JpaRepository<Order, Long> {

    @Query("""
        select o from Order o
        join fetch o.customer c
        where o.status = :status
          and c.region = :region
        order by o.createdAt desc
        """)
    List<Order> findRecentPaidOrders(@Param("status") OrderStatus status,
                                       @Param("region") String region,
                                       Pageable page);

    @Query(value = """
        select o.* from orders o
        where o.status = :status
          and o.created_at >= :since
        order by o.created_at desc
        """,
        countQuery = """
        select count(*) from orders o
        where o.status = :status
          and o.created_at >= :since
        """,
        nativeQuery = true)
    Page<Order> findByStatusSince(@Param("status") String status,
                                    @Param("since") Instant since,
                                    Pageable page);

    @Modifying
    @Transactional
    @Query("update Order o set o.status = :newStatus where o.id = :id and o.status = :oldStatus")
    int transitionStatus(@Param("id") Long id,
                          @Param("oldStatus") OrderStatus oldStatus,
                          @Param("newStatus") OrderStatus newStatus);
}
模式语法适用
JPQL实体路径 + named param大多数业务查询
Native SQLnativeQuery=true方言特性 / 复杂 join
@NamedQuery实体上 + name 引用共享查询定义
SpEL?#{#param}动态拼条件
countQuery分页时单独 SQLNative + Pageable

Native + Pageable 必须配 countQuery。JPQL 模式 Spring Data 能从 select 自动派生 count;Native 模式无法解析,需要显式 countQuery,否则抛 InvalidJpaQueryMethodException。

20.4 Projection 投影

图 20-4-1 · 3 种 Projection — DTO / Interface / Dynamic
// 1. DTO Constructor
public record OrderSummaryDto(Long id, String customerName, BigDecimal amount) {}

@Query("""
    select new com.acme.OrderSummaryDto(o.id, c.name, o.amount)
    from Order o join o.customer c
    where o.status = :status
    """)
List<OrderSummaryDto> findSummaryByStatus(@Param("status") OrderStatus status);

// 2. Interface-based (Closed Projection)
public interface OrderSummary {
    Long getId();
    String getOrderNo();
    BigDecimal getAmount();
    @Value("#{target.customer.name}")
    String getCustomerName();
}
List<OrderSummary> findByStatus(OrderStatus status);

// 3. Dynamic Projection
<T> List<T> findByStatus(OrderStatus status, Class<T> type);
// 调用: repo.findByStatus(PAID, OrderSummaryDto.class)
// 或:   repo.findByStatus(PAID, OrderSummary.class)
// 或:   repo.findByStatus(PAID, Order.class)
类型SQL 影响懒加载序列化
DTO Constructor仅 select 必要列无JSON 友好
Closed Interface仅 select 必要列无需 Jackson 配置
Open Interface (SpEL)select 全部列(加载实体)有需注意懒加载
Dynamic按 type 决定按 type灵活

20.5 分页 Pageable

@GetMapping("/orders")
public Page<Order> list(
        @RequestParam OrderStatus status,
        @PageableDefault(size = 20, sort = "createdAt", direction = DESC) Pageable page) {
    return repo.findByStatus(status, page);
}

// 接口: Page findByStatus(OrderStatus status, Pageable page);
-- 主查询
select o1_0.id, o1_0.order_no, ... from orders o1_0
where o1_0.status = ?
order by o1_0.created_at desc
offset ? rows fetch first ? rows only;
-- bind: PAID, 0, 20

-- count 查询(Page 必有,Slice 没有)
select count(o1_0.id) from orders o1_0
where o1_0.status = ?;
返回类型SQL 数含义
Page<T>主查 + count = 2有总数,可分页
Slice<T>主查 + 1(查 N+1 条)无总数,只判 hasNext
List<T>主查 1无元数据
Window<T> (3.1+)主查 + keyset游标分页,无 offset

20.6 自定义 Fragment

// 1. Fragment 接口
public interface OrderRepositoryCustom {
    List<Order> findByDynamicFilter(OrderFilter filter);
}

// 2. Custom 实现类(类名必须 = Fragment 接口名 + Impl)
@Repository
public class OrderRepositoryCustomImpl implements OrderRepositoryCustom {

    @PersistenceContext
    private EntityManager em;

    @Override
    public List<Order> findByDynamicFilter(OrderFilter filter) {
        CriteriaBuilder cb = em.getCriteriaBuilder();
        CriteriaQuery<Order> cq = cb.createQuery(Order.class);
        Root<Order> o = cq.from(Order.class);

        List<Predicate> predicates = new ArrayList<>();
        if (filter.status() != null)
            predicates.add(cb.equal(o.get("status"), filter.status()));
        if (filter.since() != null)
            predicates.add(cb.greaterThanOrEqualTo(o.get("createdAt"), filter.since()));

        cq.where(predicates.toArray(new Predicate[0]));
        return em.createQuery(cq).getResultList();
    }
}

// 3. Repository 继承 Fragment
public interface OrderRepository extends JpaRepository<Order, Long>,
                                          OrderRepositoryCustom {
}
组件命名约定职责
Fragment 接口RepoCustom自定义方法签名
实现类RepoCustomImpl(必须)Spring 自动注入
主 Repositoryextends 多个 Fragment组合而非继承

20.7 性能场景

场景反例修复收益
派生查询参数爆炸findByAAndBAndCAndD(超长方法名)@Query + Specification可读性 + 索引可控
@Query Native + Pageable 缺 countQuery分页计数错显式 countQuery正确分页
Open Projection 触发懒加载SpEL 访问关联用 Closed Projection 或 DTO无 N+1
Page 序列化包含全部实体字段API 暴露过多DTO 投影 + ResponseEntity载荷减半
findAll() 全表扫描无分页直接全量必须 Pageable / Stream / Slice避免 OOM
