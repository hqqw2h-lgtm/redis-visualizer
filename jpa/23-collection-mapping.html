第 23 章 · 集合映射与 @ElementCollection

值类型 vs 实体 · List/Set/Map · OrderColumn · Bag 退化 · 集合表批量重排

业务实体经常携带"附属数据":一个客户对应多个电话、多个地址、多个标签;一个订单对应多个里程碑、多种联系人入口。这些附属数据没有独立身份,不值得单独建实体,但又必须随宿主实体一并持久化。

JPA 规范用两条不同路径处理:@OneToMany 映射实体集合,@ElementCollection 映射值类型集合。本章从两者语义边界出发,逐一拆解 List / Set / Map 三类集合的物理模型与 Hibernate 内部行为,核心问题:修改一个集合元素,Hibernate 会发出几条 SQL?

  • 值类型(Value Type) vs 实体(Entity)8 个维度对比(@Id / 生命周期 / 共享 / 缓存 / Repository...)
  • 四类集合的 SQL 行为速查:Set / List+@OrderColumn / List(Bag) / Map
  • List 无 @OrderColumn 退化为 Bag — 删除中间元素触发 DELETE ALL + INSERT N
  • Set 的精确 SQL 依赖 equals/hashCode 的正确性
  • Map 集合的 4 种键映射:@MapKeyColumn / @MapKeyEnumerated / @MapKey / @MapKeyJoinColumn
  • @OrderBy(SQL ORDER BY,查询时排) vs @OrderColumn(持久化顺序列)
  • PersistentBag / PersistentSet / PersistentList 内部结构
  • 性能反例:每删一个标签触发 N 条 SQL,以及修复方案

前置知识:Embeddable(第 24 章前置)、equals/hashCode 契约、PersistenceContext(第 17 章)。

23.1 值类型 vs 实体类型

维度值类型(Value Type)实体(Entity)
@Id无必有
生命周期与宿主同生灭独立
引用仅一个宿主可被多处引用
equals/hashCode按所有字段值按 @Id
跨实体共享不支持支持
独立查询不可可
二级缓存随宿主缓存独立缓存区域
删除方式随宿主 / orphanRemovalDELETE 子表
Spring Data Repository不支持支持

选择标准:升级为 @OneToMany 实体,只要满足任一条件 —— ① 需要独立 @Id;② 被多个实体引用;③ 需要单独查询/分页/排序;④ 需要独立审计/权限;⑤ 集合规模 > 100 元素或增长不可控;⑥ 元素有自身状态机或业务逻辑。

23.2 集合表 DDL(PostgreSQL)

CREATE TABLE customers (
    id BIGSERIAL NOT NULL,
    email VARCHAR(180) NOT NULL,
    name VARCHAR(120) NOT NULL,
    CONSTRAINT pk_customers PRIMARY KEY (id),
    CONSTRAINT uk_customers_email UNIQUE (email)
);

-- Set 集合表:复合主键去重
CREATE TABLE customer_phones (
    customer_id BIGINT NOT NULL,
    phone VARCHAR(40) NOT NULL,
    CONSTRAINT pk_customer_phones PRIMARY KEY (customer_id, phone),
    CONSTRAINT fk_customer_phones FOREIGN KEY (customer_id)
        REFERENCES customers (id) ON DELETE CASCADE
);

-- Map 集合表:键作为复合主键的一部分
CREATE TABLE customer_settings (
    customer_id BIGINT NOT NULL,
    setting_key VARCHAR(60) NOT NULL,
    setting_value VARCHAR(255) NOT NULL,
    CONSTRAINT pk_customer_settings PRIMARY KEY (customer_id, setting_key)
);

-- List 集合表:list_index 作为顺序列
CREATE TABLE customer_addresses (
    customer_id BIGINT NOT NULL,
    list_index INTEGER NOT NULL,
    street VARCHAR(180) NOT NULL,
    city VARCHAR(80) NOT NULL,
    zip_code VARCHAR(20),
    CONSTRAINT pk_customer_addresses PRIMARY KEY (customer_id, list_index),
    CONSTRAINT ck_customer_addresses_index CHECK (list_index >= 0)
);

23.3 基本类型集合

@Entity
@Table(name = "customers")
public class Customer {

    @Id @GeneratedValue
    private Long id;

    @ElementCollection
    @CollectionTable(
        name = "customer_phones",
        joinColumns = @JoinColumn(name = "customer_id"))
    @Column(name = "phone", length = 40, nullable = false)
    private Set<String> phones = new HashSet<>();
}

-- 查询阶段
SELECT phones0_.customer_id, phones0_.phone
FROM customer_phones phones0_

23.4 List + @OrderColumn — 持久化顺序

图 23-4-1 · 删除中间元素 — @OrderColumn 触发 N 行 UPDATE 重排
@ElementCollection
@CollectionTable(name = "customer_addresses",
    joinColumns = @JoinColumn(name = "customer_id"))
@OrderColumn(name = "list_index")
private List<Address> addresses = new ArrayList<>();

List + @OrderColumn 的代价:删除中间元素会让后续元素全部 UPDATE 重排。如果业务允许乱序,改用 Set;如果需要稳定顺序但不需要按 index 访问,改用 @OrderBy。

23.5 Set — equals/hashCode 决定 SQL 精度

@Embeddable
public class Phone {
    private String countryCode;
    private String number;

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof Phone other)) return false;
        return Objects.equals(countryCode, other.countryCode)
            && Objects.equals(number, other.number);
    }

    @Override
    public int hashCode() {
        return Objects.hash(countryCode, number);
    }
}

@ElementCollection
@CollectionTable(name = "customer_phones",
    joinColumns = @JoinColumn(name = "customer_id"))
private Set<Phone> phones = new HashSet<>();

equals/hashCode 不正确会怎样?Set 会保留重复元素,Hibernate 在 flush 时找不到要删的行,可能 DELETE 所有再 INSERT 所有(行为退化为 Bag),或抛 NonUniqueObjectException。所有 Embeddable 值类型必须实现 equals/hashCode。

23.6 Map 集合 — 4 种键映射

键类型注解说明
基本类型键(String)@MapKeyColumnsetting_key VARCHAR
枚举键@MapKeyEnumeratedEnumType.STRING 强烈推荐
实体键@MapKeyJoinColumnFK 引用另一实体
实体的某属性@MapKey("name")从值实体的字段取键
 
// 基本键
@ElementCollection
@CollectionTable(name = "customer_settings",
    joinColumns = @JoinColumn(name = "customer_id"))
@MapKeyColumn(name = "setting_key", length = 60)
@Column(name = "setting_value", length = 255)
private Map<String, String> settings = new HashMap<>();

// 枚举键
@ElementCollection
@MapKeyEnumerated(EnumType.STRING)
@MapKeyColumn(name = "preference_type")
private Map<PreferenceType, String> preferences = new HashMap<>();

// 实体键(从关联实体的 id)
@OneToMany(mappedBy = "customer")
@MapKeyJoinColumn(name = "product_id")
private Map<Product, Order> ordersByProduct = new HashMap<>();

23.8 Embeddable 集合 — 复合值对象

@Embeddable
public class InventoryLocation {

    @Column(length = 180, nullable = false)
    private String street;

    @Column(length = 80, nullable = false)
    private String city;

    @Column(length = 20, nullable = false)
    private String zipCode;

    // equals/hashCode 必须实现
    @Override
    public boolean equals(Object o) { /* ... */ }
    @Override
    public int hashCode() { /* ... */ }
}

@Entity
public class Product {

    @ElementCollection
    @CollectionTable(name = "inventory_locations",
        joinColumns = @JoinColumn(name = "product_id"))
    @AttributeOverrides({
        @AttributeOverride(name = "street", column = @Column(name = "street", length = 180)),
        @AttributeOverride(name = "city",   column = @Column(name = "city", length = 80)),
        @AttributeOverride(name = "zipCode", column = @Column(name = "zip_code", length = 20))
    })
    private Set<InventoryLocation> warehouses = new HashSet<>();
}

List @OrderColumn 大批量删N 次 UPDATE 重排排序无意义 → 改 Set1 + N → 1
fetch = EAGER 大集合启动加载 N MiBLAZY + DTO 投影—
N+1 查询(LAZY 遍历)查 100 customers 触发 101 条 SQLJOIN FETCH / @BatchSize101 → 1 或 N/batch
@ElementCollection 用作日志无界增长升级为实体 + 分页—

线上事故:某电商商品标签用 List<String> tags 无 @OrderColumn,移除一个标签时 Hibernate 发出 DELETE FROM product_tags WHERE product_id=? + N 条 INSERT。商品改了 1 个标签触发 200+ 条 SQL,数据库 CPU 100%。修复:改为 Set<String> tags,SQL 数从 1+N 降到 1。

23.10 陷阱与限制

#陷阱现象修复
1List 无 @OrderColumn(Bag)删除触发 DELETE ALL + INSERT NSet 或 @OrderColumn
2Embeddable 缺 equals/hashCodeSet 行为不稳实现两方法
3@ElementCollection 默认 LAZYOSIV 触发 N+1JOIN FETCH
4大集合 EAGER启动慢/OOMLAZY 或 DTO
5Map 用 EnumType.ORDINAL枚举顺序变 → 数据错EnumType.STRING
6@OrderColumn 删除性能N 次 UPDATE评估顺序必要性
7@ElementCollection 作业务集合不能独立查询/审计升级 @OneToMany
8orphanRemoval 用在 @ElementCollection注解无效注解仅用于 @OneToMany
9关联 @Embeddable 加 @MapKey键映射混乱用 @MapKeyColumn
