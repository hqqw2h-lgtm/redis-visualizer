第 28 章 · JMX 管理与监控

v7 图解版 · 自动从 v6 markdown 生成 · Mermaid 图原样保留

JMX(Java Management Extensions,JSR-3 / JSR-160)是 Java 平台自 J2SE 5.0 起内置的标准管理框架。架构分三层:Instrumentation(MBean 定义属性、操作、通知)、Agent(MBeanServer 注册与代理)、Remote(RMI / HTTP 远程访问)。

MBean 通过 ObjectName 在 MBeanServer 中唯一标识。属性(attribute)由 getter / setter 暴露,操作(operation)由方法暴露,通知(notification)由 NotificationBroadcaster 发布。ManagementFactory.getPlatformMBeanServer() 返回 JVM 默认的平台 MBeanServer。

Spring 提供 @ManagedResource / @ManagedAttribute / @ManagedOperation 三个注解,把普通 Spring Bean 导出为 MBean。MBeanExporter 在容器初始化完成后扫描、装配、注册。Spring Boot 2.0 起 spring.jmx.enabled 默认为 false。

本章按「JMX 架构 → Spring 导出 → Actuator JMX → 远程访问 → 通知 → 陷阱 → 源码深潜」展开,聚焦运行时监控与管理操作的边界。

28.1.1 Standard MBean 示例

public interface OrderServiceMBean {
    int getPendingOrderCount();
    void setMaxConcurrentOrders(int n);
    int getMaxConcurrentOrders();
    void resetCounter();
}

public class OrderService implements OrderServiceMBean {
    private final AtomicInteger pendingCount = new AtomicInteger(0);
    private volatile int maxConcurrent = 100;

    @Override
    public int getPendingOrderCount() { return pendingCount.get(); }

    @Override
    public int getMaxConcurrentOrders() { return maxConcurrent; }

    @Override
    public void setMaxConcurrentOrders(int n) { this.maxConcurrent = n; }

    @Override
    public void resetCounter() { pendingCount.set(0); }
}

接口名以 MBean 结尾,实现类去掉 MBean 后缀,这是 JMX 反射机制的命名约定。

28.2.1 @ManagedResource 三注解

@Component
@ManagedResource(
    objectName = "com.example:type=CacheManager,name=userCache",
    description = "用户缓存管理 MBean")
public class UserCacheManager {

    private volatile int maxSize = 10000;
    private final AtomicLong hitCount = new AtomicLong(0);
    private final AtomicLong missCount = new AtomicLong(0);

    @ManagedAttribute(description = "缓存最大容量")
    public int getMaxSize() { return maxSize; }

    @ManagedAttribute(description = "设置缓存最大容量")
    public void setMaxSize(int maxSize) { this.maxSize = maxSize; }

    @ManagedAttribute(description = "缓存命中次数")
    public long getHitCount() { return hitCount.get(); }

    @ManagedAttribute(description = "缓存未命中次数")
    public long getMissCount() { return missCount.get(); }

    @ManagedAttribute(description = "缓存命中率")
    public double getHitRate() {
        long total = hitCount.get() + missCount.get();
        return total == 0 ? 0.0 : (double) hitCount.get() / total;
    }

    @ManagedOperation(description = "清空缓存并重置统计")
    public void clearCache() {
        hitCount.set(0);
        missCount.set(0);
    }

    @ManagedOperation(description = "按前缀清除缓存条目")
    @ManagedOperationParameter(name = "prefix", description = "key 前缀")
    public int evictByPrefix(String prefix) {
        return 0;
    }
}

28.5.1 NotificationBroadcasterSupport

public class OrderService extends NotificationBroadcasterSupport implements OrderServiceMBean {

    private long sequenceNumber = 0;

    public void processOrder(Order order) {
        Notification notification = new Notification(
            "order.processed",
            this,
            sequenceNumber++,
            System.currentTimeMillis(),
            "订单处理完成: " + order.getId());
        notification.setUserData(Map.of(
            "orderId", order.getId(),
            "amount", order.getAmount()));
        sendNotification(notification);
    }

    @Override
    public MBeanNotificationInfo[] getNotificationInfo() {
        return new MBeanNotificationInfo[]{
            new MBeanNotificationInfo(
                new String[]{"order.processed", "order.failed"},
                Notification.class.getName(),
                "订单处理事件通知")
        };
    }
}

28.5.2 Spring NotificationPublisher

@Component
@ManagedResource(objectName = "com.example:type=OrderService,name=main")
@ManagedNotification(
    name = "OrderNotification",
    notificationTypes = {"order.processed", "order.failed"},
    description = "订单处理事件通知")
public class OrderServiceManager implements NotificationPublisherAware {

    private NotificationPublisher notificationPublisher;
    private long sequenceNumber = 0;

    @Override
    public void setNotificationPublisher(NotificationPublisher publisher) {
        this.notificationPublisher = publisher;
    }

    @ManagedOperation(description = "处理订单")
    public void processOrder(String orderId) {
        try {
            publishNotification("order.processed", "订单处理成功: " + orderId);
        } catch (Exception e) {
            publishNotification("order.failed", "订单处理失败: " + orderId);
            throw e;
        }
    }

    private void publishNotification(String type, String message) {
        if (notificationPublisher != null) {
            Notification notification = new Notification(type, this, sequenceNumber++, message);
            notificationPublisher.sendNotification(notification);
        }
    }
}

NotificationPublisherAware 让普通 POJO 不必继承 NotificationBroadcasterSupport 即可发布 JMX 通知。Spring 在 MBeanExporter 内部把 NotificationPublisher 适配到底层 RequiredModelMBean.sendNotification()。

28.5.3 GC 通知监听

@Component
public class GcMonitor {

    private static final long PAUSE_THRESHOLD_MS = 200;

    @PostConstruct
    public void registerGcListener() {
        for (GarbageCollectorMXBean gcBean : ManagementFactory.getGarbageCollectorMXBeans()) {
            javax.management.NotificationEmitter emitter = (javax.management.NotificationEmitter) gcBean;
            emitter.addNotificationListener((notification, handback) -> {
                if (notification.getType().equals(
                        GarbageCollectionNotificationInfo.GARBAGE_COLLECTION_NOTIFICATION)) {
                    CompositeData cd = (CompositeData) notification.getUserData();
                    GarbageCollectionNotificationInfo info =
                        GarbageCollectionNotificationInfo.from(cd);
                    long duration = info.getGcInfo().getDuration();
                    if (duration > PAUSE_THRESHOLD_MS) {
                        log.warn("gc pause name={} action={} duration={}ms cause={}",
                            info.getGcName(), info.getGcAction(),
                            duration, info.getGcCause());
                    }
                }
            }, null, null);
        }
    }
}

JVM GarbageCollectorMXBean 通过 JMX 通知发布 GC 事件,监听器可获取 GC 名称、动作、暂停时间、原因。Spring Boot Actuator 的 jvm.gc.pause 指标底层使用相同机制。

28.6.1 陷阱 1:Boot 2+ 默认禁用

Spring Boot 2.0 起 spring.jmx.enabled 默认从 true 改为 false。升级时应用不报错,但所有 @ManagedResource Bean 不再注册。

spring:

@Component
public class JmxVerifier {

    @Autowired
    private MBeanServer mbs;

    @PostConstruct
    void verify() throws Exception {
        Set<ObjectName> names = mbs.queryNames(
            new ObjectName("com.example:*"), null);
        log.info("registered MBeans: {}", names);
        if (names.isEmpty()) {
            throw new IllegalStateException("no MBean registered");

设计原因:JMX 注册过程对启动时间有可测量影响(每个 MBean 约 1-2 ms),云原生场景普遍使用 HTTP 端点替代 JMX,RMI 远程访问在容器环境中存在网络问题。

28.7.2 设计取舍

取舍 1:管理接口与业务对象解耦

Spring 不要求业务 Bean 实现 DynamicMBean,而是用 RequiredModelMBean 包一层。好处是注解模型简单,业务类零侵入;代价是 MBean 元数据在启动期解析,命名冲突、大量导出会在启动期集中暴露。

取舍 2:Spring JMX 与 Actuator JMX 分层

JmxAutoConfiguration 导出 @ManagedResource Bean,JmxEndpointAutoConfiguration 把 Actuator endpoint 适配为 MBean。两条链路独立控制:业务管理操作可开,Actuator endpoint 可只暴露 health / metrics。代价是配置项分散,升级时要同时验证两个 domain。

取舍 3:JMX 适用边界

JMX 适合"可观测状态 + 低频管理操作":连接池 active / idle、清理缓存、切换只读开关。不适合高频业务查询、批量数据导出、复杂权限系统、公网管理 API。任何修改业务状态的 @ManagedOperation 必须有审计日志、幂等设计、权限隔离。

@Component
@Profile("jmx-regression")
public class JmxRegressionProbe implements ApplicationRunner {

    private final MBeanServer mBeanServer;

    public JmxRegressionProbe(MBeanServer mBeanServer) {
        this.mBeanServer = mBeanServer;
    }

    @Override
    public void run(ApplicationArguments args) throws Exception {
        ObjectName cache = new ObjectName("com.example:type=CacheManager,name=userCache");
        if (!mBeanServer.isRegistered(cache)) {
            throw new IllegalStateException("missing JMX MBean: " + cache);
        }
        Object maxSize = mBeanServer.getAttribute(cache, "MaxSize");
        if (!(maxSize instanceof Integer)) {
            throw new IllegalStateException("MaxSize must be readable integer");
        }
        MBeanInfo info = mBeanServer.getMBeanInfo(cache);
        boolean hasClearOperation = Arrays.stream(info.getOperations())
            .anyMatch(op -> op.getName().equals("clearCache"));
        if (!hasClearOperation) {
            throw new IllegalStateException("missing clearCache operation");
        }
    }
}
