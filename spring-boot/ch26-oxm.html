第 26 章 · OXM 对象 XML 映射

v7 图解版 · 自动从 v6 markdown 生成 · Mermaid 图原样保留

OXM(Object-XML Mapping)与 ORM 平行,把 Java 对象树映射到 XML 文档树。Spring 在 org.springframework.oxm 包定义 Marshaller 与 Unmarshaller 两个接口,把 JAXB、XStream、Castor 等具体实现隐藏在接口之后。

Jaxb2Marshaller 内部缓存 JAXBContext(线程安全)。每次 marshal() / unmarshal() 创建一次性的 JAXB Marshaller / Unmarshaller(非线程安全)。MarshallingHttpMessageConverter 把 OXM 接入 Spring MVC 的内容协商,MarshallingPayloadMethodProcessor 把 OXM 接入 Spring-WS 的 SOAP 消息派发。

本章按「核心抽象 → JAXB → XStream → MVC 集成 → SOAP → 陷阱 → 源码深潜」展开,聚焦企业集成场景的 XML 处理边界与失败模式。

26.1 OXM 抽象与五种实现

图 26-1-1 · Marshalling 与 Unmarshalling

26.1.1 接口签名

package org.springframework.oxm;

public interface Marshaller {
    boolean supports(Class<?> clazz);
    void marshal(Object graph, Result result) throws IOException, XmlMappingException;
}

public interface Unmarshaller {
    boolean supports(Class<?> clazz);
    Object unmarshal(Source source) throws IOException, XmlMappingException;
}

Source 与 Result 来自 javax.xml.transform,提供四种物理形态:StreamSource / StreamResult(字节流)、DOMSource / DOMResult(DOM 树)、SAXSource / SAXResult(SAX 事件)、StAXSource / StAXResult(StAX 拉式)。OXM 实现统一适配,业务代码不绑定具体形态。

表 26-1-1:OXM 五大实现对比

实现 Spring 类 配置方式 契约模型 维护状态
JAXB Jaxb2Marshaller 注解 + XSD Schema-First / Java-First 活跃,Jakarta 4.0
XStream XStreamMarshaller 反射 + 白名单 Java-First 活跃,需安全防护
Castor CastorMarshaller 映射 XML 文件 Java-First / Schema-First 已停止
JiBX JibxMarshaller 绑定文件 Schema-First 维护期
XmlBeans XmlBeansMarshaller 编译期绑定 Schema-First 维护期

26.1.2 抽象的价值

@Service
public class OrderExportService {
    private final Marshaller marshaller;
    private final Unmarshaller unmarshaller;

    public OrderExportService(Marshaller marshaller, Unmarshaller unmarshaller) {
        this.marshaller = marshaller;
        this.unmarshaller = unmarshaller;
    }

    public String exportToXml(Order order) throws IOException {
        StringWriter writer = new StringWriter();
        marshaller.marshal(order, new StreamResult(writer));
        return writer.toString();
    }

    public Order importFromXml(String xml) throws IOException {
        return (Order) unmarshaller.unmarshal(new StreamSource(new StringReader(xml)));
    }
}

@Configuration
public class OxmConfig {
    @Bean
    public Jaxb2Marshaller jaxb2Marshaller() {
        Jaxb2Marshaller marshaller = new Jaxb2Marshaller();
        marshaller.setClassesToBeBound(Order.class, OrderItem.class);
        return marshaller;
    }
}

业务代码只依赖 Marshaller / Unmarshaller,从 CastorMarshaller 切到 Jaxb2Marshaller 仅需修改 @Bean 配置。

26.2.2 核心注解

import jakarta.xml.bind.annotation.*;

@XmlRootElement(name = "order", namespace = "http://example.com/orders")
@XmlAccessorType(XmlAccessType.FIELD)
@XmlType(propOrder = {"orderId", "customer", "orderDate", "totalAmount", "items"})
public class Order {

    @XmlAttribute(name = "id", required = true)
    private Long orderId;

    @XmlElement(required = true)
    private String customer;

    @XmlElement(name = "order-date")
    @XmlSchemaType(name = "date")
    private LocalDate orderDate;

    @XmlElement(name = "total")
    private BigDecimal totalAmount;

    @XmlElementWrapper(name = "items")
    @XmlElement(name = "item")
    private List<OrderItem> items = new ArrayList<>();

    @XmlTransient
    private String internalNotes;

    public Order() {}
}

输出形态由注解直接决定:@XmlAttribute 落到属性位置,@XmlElementWrapper 增加包装层,@XmlTransient 排除字段。

<order xmlns="http://example.com/orders" id="1001">
  <customer>Customer C1001 Wang</customer>
  <order-date>2026-04-20</order-date>
  <total>599.00</total>
  <items>
    <item sku="LAPTOP-01" quantity="1">
      <price>599.00</price>
    </item>
  </items>
</order>

表 26-2-1:@XmlAccessorType 四种策略

策略 绑定来源 优点 缺点
FIELD 所有非 static、非 transient 字段 直观、JPA 友好 需 @XmlTransient 排除
PROPERTY getter / setter 对 可在 getter 加转换 容易重复绑定

NONE 仅显式标注的字段 / 属性 最精确 注解负担重

26.2.3 绑定编译

图 26-2-3 · JAXB 绑定编译双路径(Schema-First xjc / Java-First schemagen → 共用 JAXBContext)

JAXBContext.newInstance() 扫描注解、构建元数据、缓存,开销在毫秒至秒级。createMarshaller() / createUnmarshaller() 创建非线程安全的实例,微秒级。错误的做法是每次请求新建 JAXBContext,正确做法是 Jaxb2Marshaller 单例缓存。

26.2.4 直接调用 JAXB

public class JaxbExample {
    private static final JAXBContext CONTEXT;
    static {
        try {
            CONTEXT = JAXBContext.newInstance(Order.class, OrderItem.class);
        } catch (JAXBException e) {
            throw new ExceptionInInitializerError(e);
        }
    }

    public String marshal(Order order) throws JAXBException {
        jakarta.xml.bind.Marshaller marshaller = CONTEXT.createMarshaller();
        marshaller.setProperty(jakarta.xml.bind.Marshaller.JAXB_FORMATTED_OUTPUT, true);
        marshaller.setProperty(jakarta.xml.bind.Marshaller.JAXB_ENCODING, "UTF-8");
        StringWriter writer = new StringWriter();
        marshaller.marshal(order, writer);
        return writer.toString();
    }

    public Order unmarshal(String xml) throws JAXBException {
        jakarta.xml.bind.Unmarshaller unmarshaller = CONTEXT.createUnmarshaller();
        return (Order) unmarshaller.unmarshal(new StringReader(xml));
    }
}

26.2.5 Schema 验证

public Order unmarshalWithValidation(String xml) throws JAXBException, SAXException {
    jakarta.xml.bind.Unmarshaller unmarshaller = CONTEXT.createUnmarshaller();
    SchemaFactory sf = SchemaFactory.newInstance(XMLConstants.W3C_XML_SCHEMA_NS_URI);
    Schema schema = sf.newSchema(getClass().getResource("/schemas/order.xsd"));
    unmarshaller.setSchema(schema);
    unmarshaller.setEventHandler(event -> {
        if (event.getSeverity() == ValidationEvent.ERROR ||
            event.getSeverity() == ValidationEvent.FATAL_ERROR) {
            throw new RuntimeException("XML validation failed: " + event.getMessage());
        }
        return true;
    });
    return (Order) unmarshaller.unmarshal(new StringReader(xml));
}

校验失败时 ValidationEventHandler 把 cvc-* 错误转成业务可见异常,并在入口拦截非法报文,避免污染下游业务逻辑。

26.2.6 XmlAdapter 自定义类型转换

public class LocalDateAdapter extends XmlAdapter<String, LocalDate> {
    private static final DateTimeFormatter FMT = DateTimeFormatter.ISO_LOCAL_DATE;

    @Override
    public LocalDate unmarshal(String v) {
        return v != null ? LocalDate.parse(v, FMT) : null;
    }

    @Override
    public String marshal(LocalDate v) {
        return v != null ? v.format(FMT) : null;
    }
}

@XmlElement(name = "order-date")
@XmlJavaTypeAdapter(LocalDateAdapter.class)
private LocalDate orderDate;

包级别全局注册:

@XmlJavaTypeAdapters({
    @XmlJavaTypeAdapter(value = LocalDateAdapter.class, type = LocalDate.class),
    @XmlJavaTypeAdapter(value = LocalDateTimeAdapter.class, type = LocalDateTime.class),
    @XmlJavaTypeAdapter(value = BigDecimalAdapter.class, type = BigDecimal.class)
})
package com.example.order.model;

import jakarta.xml.bind.annotation.adapters.*;

XmlAdapter 仅做格式转换,不应包含业务逻辑、数据库查询或远程调用。其职责边界是「Java 类型 ↔ XML 文本表示」。

26.3.3 安全:类型白名单

XStream 1.4.17 及更早版本存在 14 个反序列化漏洞(CVE-2021-39139 至 CVE-2021-39153)。攻击者构造恶意 XML 可在反序列化时执行任意代码(RCE)。XStream 1.4.18+ 默认拒绝所有类型,必须显式配置白名单。

XStream xstream = new XStream();
xstream.allowTypes(new Class[]{Order.class, OrderItem.class, Customer.class});
xstream.allowTypesByWildcard(new String[]{"com.example.model.**"});
xstream.allowTypesByRegExp(new String[]{"com\\.example\\..*"});

XStream 默认禁止的危险类型包括 java.lang.ProcessBuilder、java.lang.Runtime、javax.script.*、java.io.*。

com.thoughtworks.xstream.security.ForbiddenClassException: java.lang.ProcessBuilder

表 26-3-1:JAXB 与 XStream 选型

维度 JAXB XStream
配置方式 注解 + XSD 反射 + 别名
契约强度 强(XSD 校验) 弱(无 schema)
跨语言 是(XSD 通用) 否(Java 类绑定)
安全 仅绑定指定类型 默认拒绝,需白名单
性能 缓存 JAXBContext 后较快 反射开销大
适用场景 对外契约 / SOAP 内部配置 / 测试夹具

26.4 Spring OXM 与 MVC 集成

图 26-4-1 · HTTP 内容协商

26.4.1 配置 Jaxb2Marshaller

@Configuration
public class OxmConfiguration {
    @Bean
    public Jaxb2Marshaller jaxb2Marshaller() {
        Jaxb2Marshaller marshaller = new Jaxb2Marshaller();
        marshaller.setClassesToBeBound(Order.class, OrderItem.class, Customer.class);
        marshaller.setSchema(new ClassPathResource("schemas/order.xsd"));
        Map<String, Object> props = new HashMap<>();
        props.put(jakarta.xml.bind.Marshaller.JAXB_FORMATTED_OUTPUT, true);
        props.put(jakarta.xml.bind.Marshaller.JAXB_ENCODING, "UTF-8");
        marshaller.setMarshallerProperties(props);
        return marshaller;
    }
}

Jaxb2Marshaller 在 afterPropertiesSet() 中创建并缓存 JAXBContext。启动日志应仅出现一次「Creating JAXBContext」,而非每个请求重复创建。

26.4.2 注册 MarshallingHttpMessageConverter

@Configuration
public class WebMvcOxmConfig implements WebMvcConfigurer {

    @Autowired
    private Jaxb2Marshaller jaxb2Marshaller;

    @Override
    public void configureMessageConverters(List<HttpMessageConverter<?>> converters) {
        MarshallingHttpMessageConverter xmlConverter =
            new MarshallingHttpMessageConverter(jaxb2Marshaller);
        xmlConverter.setSupportedMediaTypes(List.of(
            MediaType.APPLICATION_XML, MediaType.TEXT_XML));
        converters.add(xmlConverter);
        converters.add(new MappingJackson2HttpMessageConverter());
    }

    @Override
    public void configureContentNegotiation(ContentNegotiationConfigurer configurer) {
        configurer
            .defaultContentType(MediaType.APPLICATION_JSON)
            .favorParameter(true)
            .parameterName("format")
            .mediaType("json", MediaType.APPLICATION_JSON)
            .mediaType("xml", MediaType.APPLICATION_XML);
    }
}

26.5.4 @Endpoint 实现

@Endpoint
public class OrderEndpoint {

    private static final String NAMESPACE_URI = "http://example.com/orders";

    @Autowired
    private OrderService orderService;

    @PayloadRoot(namespace = NAMESPACE_URI, localPart = "GetOrderRequest")
    @ResponsePayload
    public GetOrderResponse getOrder(@RequestPayload GetOrderRequest request) {
        Order order = orderService.findById(request.getOrderId());
        OrderType orderType = new OrderType();
        orderType.setOrderId(order.getId());
        orderType.setCustomer(order.getCustomer());
        orderType.setTotal(order.getTotal());
        orderType.setStatus(OrderStatus.fromValue(order.getStatus().name()));
        GetOrderResponse response = new GetOrderResponse();
        response.setOrder(orderType);
        return response;
    }

    @PayloadRoot(namespace = NAMESPACE_URI, localPart = "CreateOrderRequest")
    @ResponsePayload
    public CreateOrderResponse createOrder(@RequestPayload CreateOrderRequest request) {
        Order order = orderService.create(mapToOrder(request));
        CreateOrderResponse response = new CreateOrderResponse();
        response.setOrderId(order.getId());
        response.setStatus("CREATED");
        return response;
    }
}

    }
}

marshalSendAndReceive() 内部完成「Java → XML(marshal)→ HTTP POST → XML → Java(unmarshal)」全过程。客户端代码看到的只是对象传入与对象返回。

26.6 陷阱与限制

表 26-6-1:陷阱明细

# 陷阱 症状 根因 修复

26.6.1 陷阱 1:JAXB 从 JDK 11 移除

// 反例:JDK 8 时代未声明依赖
// 升级到 JDK 11+ 立即报 ClassNotFoundException

// 正例:Spring Boot 3.x 显式声明
// pom.xml:
//   jakarta.xml.bind:jakarta.xml.bind-api
//   org.glassfish.jaxb:jaxb-runtime

包名迁移表:

Spring Boot JDK JAXB API 包名
2.x 8 / 11 / 17 javax.xml.bind:jaxb-api javax.xml.bind.*
3.x 17+ jakarta.xml.bind:jakarta.xml.bind-api jakarta.xml.bind.*

迁移时必须同步更新所有 import 语句。Spring Boot 3.x 的 Jaxb2Marshaller 仅支持 Jakarta 命名空间。

26.6.2 陷阱 2:XStream 安全

// 反例:默认信任所有类型
XStream xstream = new XStream();
Object result = xstream.fromXML(untrustedXml);
// 攻击者构造 <java.lang.ProcessBuilder> 即可执行系统命令

// 正例:1.4.18+ 强制白名单
XStream xstream = new XStream();
xstream.addPermission(NoTypePermission.NONE);
xstream.addPermission(NullPermission.NULL);
xstream.addPermission(PrimitiveTypePermission.PRIMITIVES);
xstream.allowTypes(new Class[]{Order.class, OrderItem.class});

XStream 在外部输入场景需配合 XXE 防护:XppDriver 默认不解析外部实体,但 DomDriver 在某些 JDK 版本启用了 DOCTYPE 处理,需通过 XMLConstants.FEATURE_SECURE_PROCESSING 关闭。

26.6.3 陷阱 3:命名空间冲突

// 反例:同 JAXBContext 包含两个 root local name 相同的类
@XmlRootElement(name = "order", namespace = "http://purchase.example.com")
public class PurchaseOrder { }

@XmlRootElement(name = "order", namespace = "http://sales.example.com")
public class SalesOrder { }

// JAXBContext.newInstance(PurchaseOrder.class, SalesOrder.class)
// → IllegalAnnotationsException: Two classes have the same XML type name "order"

// 正例:用 @XmlType 区分 XML type name
@XmlRootElement(name = "order", namespace = "http://purchase.example.com")
@XmlType(name = "PurchaseOrderType", namespace = "http://purchase.example.com")
public class PurchaseOrder { }

@XmlRootElement(name = "order", namespace = "http://sales.example.com")
@XmlType(name = "SalesOrderType", namespace = "http://sales.example.com")
public class SalesOrder { }

或者使用独立的 Jaxb2Marshaller,每个上下文只绑定一组类。

26.6.4 陷阱 4:循环引用

// 反例
@XmlRootElement
public class Order {
    @XmlElement
    private Customer customer;
}

@XmlRootElement
public class Customer {
    @XmlElement
    private List<Order> orders;
}
// marshal(order) → StackOverflowError

// 方案 1:@XmlTransient 断开
public class Customer {
    @XmlTransient
    private List<Order> orders;
}

// 方案 2:@XmlIDREF 引用而非内嵌
public class Order {
    @XmlIDREF
    @XmlElement
    private Customer customer;
}
public class Customer {
    @XmlID
    @XmlAttribute
    private String customerId;
    @XmlTransient
    private List<Order> orders;
}

XStream 启用引用模式:xstream.setMode(XStream.ID_REFERENCES),通过 id / reference 属性表示循环。

26.6.5 陷阱 5:性能差距

+----------------------+----------------+----------------+
| 数据格式             | 体积(bytes)   | parse p95     |
+----------------------+----------------+----------------+
| JSON                 | 156            | 0.8 ms         |
| XML(无 namespace)   | 234            | 2.1 ms         |
| XML + namespace      | 298            | 2.8 ms         |
| XML + Schema 验证    | 298            | 4.5 ms         |
+----------------------+----------------+----------------+

XML 体积约为同等数据 JSON 的 1.5-2 倍,DOM 解析的内存开销和 SAX / StAX 事件处理的 CPU 开销均高于 JSON 流式解析。

优化:大文件优先 StreamSource / StreamResult 或 StAXSource / StAXResult,避免先构建 DOM;对大型 XML 响应启用 GZIP 压缩(server.compression.enabled=true,server.compression.mime-types=application/xml)。

26.6.6 陷阱 6:Schema 演进

破坏性变更:把 <phone> 从 xs:string 改为复杂类型,旧客户端的 <phone>+86-10-12345678</phone> 立即验证失败。

安全的 XSD 演进规则:

1. 仅添加 minOccurs="0" 的可选元素
2. 放宽约束(maxLength 增大、enum 新增值)
3. 使用版本化命名空间(http://example.com/orders/v1 → /v2)
4. WSDL 同时暴露 v1 与 v2 端点

Spring-WS 多版本 Endpoint:

@Endpoint
public class OrderEndpoint {
    @PayloadRoot(namespace = "http://example.com/orders/v1", localPart = "GetOrderRequest")
    @ResponsePayload
    public GetOrderResponseV1 getOrderV1(@RequestPayload GetOrderRequestV1 request) { }

    @PayloadRoot(namespace = "http://example.com/orders/v2", localPart = "GetOrderRequest")
    @ResponsePayload
    public GetOrderResponseV2 getOrderV2(@RequestPayload GetOrderRequestV2 request) { }
}

26.7.1 设计取舍

取舍 1:契约由 XSD 控制还是 Java 注解控制

Schema-First 把契约权交给 XSD,用 xjc 生成 Java 类,业务代码无法绕过契约;Java-First 简化开发但容易把内部模型泄漏到对外契约。对外 SOAP / 银行报文走 Schema-First,内部工具或测试可走 Java-First。

取舍 2:抽象成本与切换收益

Marshaller / Unmarshaller 接口的抽象成本是接口跳转与 @Bean 配置;收益是 OXM 实现切换不需修改业务代码。在 Castor 停止维护后切换到 JAXB 的项目验证了该收益。

取舍 3:验证开销与防御深度

XSD 验证增加 30%-50% unmarshal 耗时,但能在入口拦截非法报文,避免污染业务层。外部输入场景默认开启 XSD 验证,内部可信通道按风险降级。

@SpringBootTest
class OxmIntegrationTests {

    @Autowired MockMvc mockMvc;

        assertThat(order.getOrderId()).isEqualTo(1001L);
        assertThat(order.getItems()).hasSize(1);

            .andExpect(content().contentType(MediaType.APPLICATION_XML))

26.8 本章要点

要点 源码 / 配置依据 易错点
Marshaller / Unmarshaller 是 Spring OXM 核心抽象 org.springframework.oxm 包 业务代码只依赖接口
Jaxb2Marshaller 缓存 JAXBContext,线程安全 afterPropertiesSet() 不要每请求新建上下文
JAXB Marshaller / Unmarshaller 实例非线程安全 createMarshaller() 每次调用创建
Spring Boot 3.x 使用 Jakarta 命名空间 jakarta.xml.bind.* 与 Spring Boot 2.x 不兼容
XStream 1.4.18+ 默认拒绝所有类型 addPermission / allowTypes 无白名单则反序列化失败
MarshallingHttpMessageConverter 接入 MVC 内容协商 configureMessageConverters Accept 头驱动选择
MarshallingPayloadMethodProcessor 接入 SOAP Body MessageDispatcher 仅处理 Body,不处理 Envelope
@PayloadRoot 路由按 namespace + localPart PayloadRootAnnotationMethodEndpointMapping 同 localName 不同 namespace 视为不同 endpoint
Schema 验证失败应拦截在入口 ValidationEventHandler 不要污染业务层
大 XML 文件优先 Stream / StAX StreamSource / StAXSource DOM 内存放大数倍
XSD 演进必须保持兼容 minOccurs="0" / 版本化 namespace 简单类型改复杂类型必破坏兼容
XmlAdapter 仅做格式转换 XmlAdapter<XmlType, JavaType> 不应包含业务逻辑
