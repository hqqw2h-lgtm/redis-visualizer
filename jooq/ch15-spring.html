第 15 章 · Spring Boot 集成

v7 图解版 · 自动从 v6 markdown 生成 · Mermaid 图原样保留

jOOQ 自身是一个纯 JDBC 库,不依赖容器。但生产项目大量运行在 Spring Boot 之上。spring-boot-starter-jooq 通过 JooqAutoConfiguration 自动装配 DataSource → ConnectionProvider → DSLContext,并通过 TransactionAwareDataSourceProxy 让 jOOQ 与 Spring @Transactional 共享同一 JDBC Connection,实现"注解一打、jOOQ 自动参与事务"的无缝体验。

本章按"自动配置 → 事务集成 → 自定义 → JPA 共存 → 测试 → R2DBC → 陷阱 → 守护 → 源码深潜"展开。Spring Boot 集成的关键不在 DSLContext Bean 本身,而在它背后的 Configuration 被谁创建、连接从哪里来、事务上下文如何被发现、异常如何转换。

15.1.2 装配链

图 15-1-2 · 装配链 — application.yml → DataSource → Auto → Cfg → DSLContext

15.1.3 自动装配伪代码

@Configuration
@ConditionalOnClass(DSLContext.class)
@AutoConfigureAfter(DataSourceAutoConfiguration.class)
public class JooqAutoConfiguration {

    @Bean @ConditionalOnMissingBean
    public SQLDialect jooqSqlDialect(DataSource ds) throws SQLException {
        try (Connection c = ds.getConnection()) {
            return JDBCUtils.dialect(c.getMetaData().getURL());
        }
    }

    @Bean @ConditionalOnMissingBean
    public ConnectionProvider connectionProvider(DataSource ds) {
        return new DataSourceConnectionProvider(
            new TransactionAwareDataSourceProxy(ds));
    }

    @Bean @ConditionalOnMissingBean
    public TransactionProvider transactionProvider(PlatformTransactionManager tm) {
        return new SpringTransactionProvider(tm);
    }

    @Bean @ConditionalOnMissingBean
    public DefaultDSLContext dslContext(Configuration cfg) {
        return new DefaultDSLContext(cfg);
    }
}

15.2 事务集成

图 15-2:@Transactional + jOOQ 时序

图 15-2 · @Transactional + jOOQ 时序 — ThreadLocal 共享 Connection

15.2.1 @Transactional 示例

@Service
public class OrderService {
    private final DSLContext dsl;

    public OrderService(DSLContext dsl) { this.dsl = dsl; }

    @Transactional
    public void createOrder(OrderDTO dto) {
        OrderRecord order = dsl.insertInto(ORDER)
            .set(ORDER.CUSTOMER_ID, dto.customerId())
            .set(ORDER.AMOUNT, dto.amount())
            .returning(ORDER.ID)
            .fetchOne();

        for (ItemDTO item : dto.items()) {
            dsl.insertInto(ORDER_ITEM)
               .set(ORDER_ITEM.ORDER_ID, order.getId())
               .set(ORDER_ITEM.PRODUCT_ID, item.productId())
               .set(ORDER_ITEM.QUANTITY, item.quantity())
               .execute();
        }
    }
}

PostgreSQL SQL 序列:

begin;
insert into "order" ("customer_id", "amount") values (?, ?) returning "id";
insert into "order_item" ("order_id", "product_id", "quantity") values (?, ?, ?);
commit;

Spring Boot 中应优先使用 @Transactional 作为唯一事务入口。Boot 已把 jOOQ 事务委托给 Spring,在 @Transactional 内再嵌套 dsl.transaction() 会产生不直观的传播 / Savepoint 语义。

表 15-2:传播行为

传播 jOOQ Connection 行为 典型场景
REQUIRED(默认) 共享同一 Conn 大多数业务方法
REQUIRES_NEW 新 Conn,挂起当前 审计日志(独立提交)
NESTED 同一 Conn + Savepoint 部分回滚
NOT_SUPPORTED 挂起,autoCommit 只读报表

@Repository
public class CustomerRepository {
    private final DSLContext dsl;
    public CustomerRepository(DSLContext dsl) { this.dsl = dsl; }

    public void createCustomer(long id, String name) {

唯一键冲突 → Spring DuplicateKeyException:

org.springframework.dao.DuplicateKeyException:

业务层无需判断 PostgreSQL / MySQL / H2 各自不同的厂商错误码。

15.3 自定义 Configuration

图 15-3:两条扩展路径

图 15-3 · 自定义 Configuration 两条路径 — Customizer 追加 vs DSLContext Bean 替换

15.3.1 推荐:DefaultConfigurationCustomizer

@Configuration
public class JooqConfig {

    @Bean
    public DefaultConfigurationCustomizer jooqCustomizer() {
        return cfg -> {
            ExecuteListenerProvider[] existing = cfg.executeListenerProviders();
            ExecuteListenerProvider[] extra = {
                new DefaultExecuteListenerProvider(new SqlLogListener()),
                new DefaultExecuteListenerProvider(
                    new SlowQueryListener(Duration.ofSeconds(3)))
            };
            cfg.set(Stream.concat(Arrays.stream(existing), Arrays.stream(extra))
                          .toArray(ExecuteListenerProvider[]::new));

            cfg.set(new Settings()
                .withRenderFormatted(true)
                .withExecuteLogging(false)
                .withMapConstructorParameterNamesInKotlin(true));

            cfg.set(new CustomRecordMapperProvider());
        };
    }
}

15.3.2 完全自定义 @Bean

@Configuration
public class JooqManualConfig {

    @Bean
    public DSLContext dslContext(DataSource ds, PlatformTransactionManager tm) {
        DefaultConfiguration cfg = new DefaultConfiguration();
        cfg.set(SQLDialect.POSTGRES);
        cfg.set(new DataSourceConnectionProvider(
            new TransactionAwareDataSourceProxy(ds)));
        cfg.set(new SpringTransactionProvider(tm));

        cfg.set(new Settings()
            .withRenderSchema(false)
            .withRenderFormatted(true));

        cfg.set(
            new DefaultExecuteListenerProvider(new SqlLogListener()),
            new DefaultExecuteListenerProvider(new MetricsListener()));

        return new DefaultDSLContext(cfg);
    }
}

手动声明 DSLContext Bean 时必须包 TransactionAwareDataSourceProxy,否则 jOOQ 每次拿池中新连接,@Transactional 形同虚设。

15.4 jOOQ + JPA 共存

图 15-4:共存架构

图 15-4 · JPA + jOOQ 共存架构 — 同一 @Transactional + 同一 Connection

15.4.1 混用示例

@Service
public class OrderAnalyticsService {
    private final OrderRepository orderRepo;
    private final DSLContext dsl;

    @Transactional
    public OrderStatsDTO enrichAndQuery(Long orderId) {
        OrderEntity order = orderRepo.findById(orderId).orElseThrow();
        order.setLastAccessed(Instant.now());
        orderRepo.save(order);

        orderRepo.flush();  // 关键:JPA 一级缓存写入 DB,jOOQ 才能读到

        return dsl.select(
                ORDERS.CUSTOMER_ID,
                count().as("order_count"),
                avg(ORDERS.AMOUNT).as("avg_amount"),
                sum(ORDERS.AMOUNT).as("total_amount"))
            .from(ORDERS)
            .where(ORDERS.CUSTOMER_ID.eq(order.getCustomerId()))
            .groupBy(ORDERS.CUSTOMER_ID)
            .fetchOneInto(OrderStatsDTO.class);
    }
}

表 15-4:工具选型

场景 推荐 原因
单表 CRUD JPA 自动脏检查 / 级联
多表 JOIN + 窗口 jOOQ 类型安全 / 方言 / 表达力
动态多条件 jOOQ Condition API 简洁
批量(>1000) jOOQ Batch / Loader 远优于 JPA batch
实体关系图 JPA 懒加载 / 级联

JPA flush 陷阱:Hibernate 默认延迟写入,jOOQ 直接读 DB 可能看不到刚写入但未 flush 的数据。混用方法必须先 flush()。

Spring Boot 在 JPA + jOOQ 共存时默认使用 JpaTransactionManager,该 manager 同时支持 JPA 和 JDBC 操作,无需额外配置。

class OrderServiceMockTest {
    @Test
    void should_calculate_total_revenue() {
        MockDataProvider provider = ctx -> {

            return new MockResult[]{ new MockResult(0, null) };
        };

        assertThat(total).isEqualByComparingTo("79.98");

}

15.6 R2DBC 与响应式

图 15-6:JDBC vs R2DBC

图 15-6 · JDBC 阻塞 vs R2DBC 响应式 — 两套独立装配路径

15.6.1 配置

@Configuration
public class JooqR2dbcConfig {

    @Bean
    public ConnectionFactory connectionFactory() {
        return ConnectionFactories.get(
            ConnectionFactoryOptions.builder()
                .option(DRIVER, "postgresql")
                .option(HOST, "localhost")
                .option(PORT, 5432)
                .option(DATABASE, "mydb")
                .option(USER, "app")
                .option(PASSWORD, "secret")
                .build());
    }

    @Bean
    public DSLContext dslContext(ConnectionFactory cf) {
        return DSL.using(cf, SQLDialect.POSTGRES);
    }
}

15.6.2 响应式查询

@Service
public class ReactiveOrderService {
    private final DSLContext dsl;

    public Flux<OrdersRecord> findExpensiveOrders(BigDecimal min) {
        return Flux.from(
            dsl.selectFrom(ORDERS)
               .where(ORDERS.AMOUNT.gt(min))
               .orderBy(ORDERS.AMOUNT.desc()));
    }

    public Mono<Integer> updatePrice(Long id, BigDecimal price) {
        return Mono.from(
            dsl.update(ORDERS)
               .set(ORDERS.AMOUNT, price)
               .where(ORDERS.ID.eq(id)));
    }

    public Flux<OrdersRecord> streamAll() {
        return Flux.from(dsl.selectFrom(ORDERS))
            .onBackpressureBuffer(256);
    }
}

R2DBC 事务不能用 DataSourceTransactionManager。必须用 ReactiveTransactionManager 或 TransactionalOperator,事务上下文沿 Reactor Context 传播。jOOQ 也提供 dsl.transactionPublisher()。

15.10 源码深潜与本章要点

表 15-10:源码入口

入口 包 / 类 职责

设计取舍

取舍 1:TransactionAwareDataSourceProxy 而非 jOOQ 自己感知线程

Spring 的事务管理基于 ThreadLocal,jOOQ 通过代理 DataSource.getConnection() 委托给 Spring。代价是必须包裹 Proxy,收益是 jOOQ 与 JPA / JdbcTemplate 等所有数据访问技术共享同一连接。

取舍 2:DefaultConfigurationCustomizer 而非完全替换 Bean

让用户在 Boot 默认配置基础上追加,而非覆盖。该设计保留异常转换、连接代理、事务委托等 Boot 内置能力,只允许业务侧关注的 listener / settings / mapper 被定制。

取舍 3:不为 R2DBC 提供官方 Starter

R2DBC 生态尚未稳定,Spring Boot 暂不为 jOOQ R2DBC 提供 Starter。用户必须显式声明 ConnectionFactory 和 DSLContext,代价是配置繁琐,但避免 Boot 锁定一种 R2DBC 接入方式。

取舍 4:JPA 与 jOOQ 共享 JpaTransactionManager 而非各用一个

JpaTransactionManager 同时管理 JPA EntityManager 与底层 JDBC Connection,确保两者共享事务。若各用一个 manager,同一业务方法可能拿到不同连接,flush / 锁 / 回滚语义崩坏。

取舍 5:@JooqTest 切片仅加载 jOOQ 相关 Bean

切片测试不加载 @Controller / @Service,启动远快于 @SpringBootTest。代价是 Service 层依赖必须 @Import 显式导入;收益是 Repository 层快速回归。
