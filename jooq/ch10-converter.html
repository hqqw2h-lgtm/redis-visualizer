第 10 章 · Converter 与 Binding

v7 图解版 · 自动从 v6 markdown 生成 · Mermaid 图原样保留

数据库存储的物理类型(INTEGER / VARCHAR / JSONB / ENUM)与业务领域的 Java 类型(Boolean / OrderStatus / CustomerProfile)之间存在结构性鸿沟。jOOQ 通过 Converter<T, U> 与 Binding<T, U> 两个 SPI 把该鸿沟关闭:前者负责"值转换",后者负责"JDBC 交互全控制"。两者绑定在 DataType<T> 与代码生成的 TableField<U> 上,使类型转换覆盖查询的双向链路:where(STATUS.eq(enum)) 写入参数与 fetch() 读取结果都自动转换。

本章按"源码入口 → Converter → 常用场景 → Binding → ForcedType → 枚举三策略 → 陷阱 → 生产守卫"展开。生产视角是本章重点:Converter 不是"转一下值"的工具类,而是数据库边界的一部分。一次 from() 失败可能让查询已成功的 fetch 阶段崩溃;一次 to() 失败可能让批量写入回滚。稳定的 Converter 需明确 null 策略、未知值策略、类型漂移策略与回归测试。

10.2.3 JSONB ↔ Java 对象

图 10-2-3 · JSONB ↔ Java 对象 — 序列化与反序列化双向流
public class JsonConverter<U> implements Converter<JSONB, U> {
    private static final ObjectMapper OM = new ObjectMapper();
    private final Class<U> toType;

    public JsonConverter(Class<U> toType) { this.toType = toType; }

    @Override public U from(JSONB db) {
        if (db == null || db.data() == null) return null;
        try { return OM.readValue(db.data(), toType); }
        catch (Exception e) { throw new DataTypeException("JSON parse error", e); }
    }
    @Override public JSONB to(U user) {
        if (user == null) return null;
        try { return JSONB.jsonb(OM.writeValueAsString(user)); }
        catch (Exception e) { throw new DataTypeException("JSON write error", e); }
    }
    @Override public Class<JSONB> fromType() { return JSONB.class; }
    @Override public Class<U>     toType()   { return toType; }
}

10.2.4 加密字段

Converter<String, String> encrypt = Converter.ofNullable(
    String.class, String.class,
    cipher -> AesUtil.decrypt(cipher, kms.fetchKey()),
    plain  -> AesUtil.encrypt(plain, kms.fetchKey()));

密钥应从 KMS / Vault 获取,不得硬编码或存入 Git。Converter 仅作为加密层入口,加密算法、密钥轮换、审计应在外部密钥管理服务侧实现。

10.3.2 Binding 调用时机

图 10-3-2 · Binding 调用时序 — sql / set / get 协同 JDBC

10.3.3 PostgreSQL JSONB Binding

public class PostgresJsonbBinding extends AbstractBinding<JSONB, CustomerProfile> {
    private final JsonConverter<CustomerProfile> converter =
        new JsonConverter<>(CustomerProfile.class);

    @Override public Converter<JSONB, CustomerProfile> converter() {
        return converter;
    }

    @Override public void sql(BindingSQLContext<CustomerProfile> ctx) {
        ctx.render()
           .visit(DSL.val(ctx.convert(converter).value(), JSONB.class))
           .sql("::jsonb");
    }

    @Override public void set(BindingSetStatementContext<CustomerProfile> ctx)
            throws SQLException {
        JSONB jsonb = converter.to(ctx.value());
        if (jsonb == null) {
            ctx.statement().setNull(ctx.index(), Types.OTHER);
            return;
        }
        PGobject pg = new PGobject();
        pg.setType("jsonb");
        pg.setValue(jsonb.data());
        ctx.statement().setObject(ctx.index(), pg);
    }

    @Override public void get(BindingGetResultSetContext<CustomerProfile> ctx)
            throws SQLException {
        String json = ctx.resultSet().getString(ctx.index());
        ctx.value(converter.from(json == null ? null : JSONB.jsonb(json)));
    }

    @Override public void register(BindingRegisterContext<CustomerProfile> ctx)
            throws SQLException {
        ctx.statement().registerOutParameter(ctx.index(), Types.OTHER);
    }
}

Binding.sql() 是 Converter 无法替代的关键能力:它注入自定义 SQL 片段(如 ::jsonb),而 PostgreSQL JDBC 驱动不会自动将 String 转换为 jsonb 类型。

10.4.3 配置元素对照

    Converter<?, ?> converter = status.getDataType().getConverter();

@Test
void forcedType_converts_bind_value_when_rendering() {
    DSLContext dsl = DSL.using(SQLDialect.POSTGRES);
    Query q = dsl.insertInto(ORDERS)
        .set(ORDERS.ID, 1L)
        .set(ORDERS.STATUS, OrdersStatus.PUBLISHED);

    assertThat(q.getBindValues()).contains(OrdersStatus.PUBLISHED);

includeExpression 必须匹配全限定路径 catalog.schema.table.column,而非仅列名。仅写 status 不会匹配,需 .*\.status 锚定。

10.5.5 策略 C:INTEGER + 显式 code

public enum OrderState {
    NEW(10), PAID(20), SHIPPED(30), CANCELLED(40);

    private final int code;

    OrderState(int code) { this.code = code; }
    public int code() { return code; }

    public static OrderState fromCode(int code) {
        for (OrderState s : values()) {
            if (s.code == code) return s;
        }
        throw new DataTypeException("Unknown order_state code: " + code);
    }
}

public final class OrderStateConverter implements Converter<Integer, OrderState> {
    @Override public OrderState from(Integer db) {
        return db == null ? null : OrderState.fromCode(db);
    }
    @Override public Integer to(OrderState user) {
        return user == null ? null : user.code();
    }
    @Override public Class<Integer> fromType() { return Integer.class; }
    @Override public Class<OrderState> toType() { return OrderState.class; }
}

ordinal() 是脆弱的:任何插入新值或重排顺序都会破坏映射。生产代码必须使用显式 code 字段,即使遗留系统使用 INTEGER 存储。

# 陷阱 现象 根因 修复

关键反例与正例

// 反例:plain SQL 不绑定 Converter
dsl.resultQuery("SELECT status FROM orders")
   .fetch();
// 返回 String 而非 OrdersStatus

// 正例:codegen 字段
dsl.select(ORDERS.STATUS).from(ORDERS).fetch();

// 正例:显式 DataType
Field<OrdersStatus> STATUS = DSL.field(
    "status",
    SQLDataType.VARCHAR.asConvertedDataType(statusConverter));
dsl.select(STATUS).from("orders").fetch();

// 反例:脆弱的枚举 Converter
Converter.ofNullable(String.class, OrdersStatus.class,
    OrdersStatus::valueOf,
    OrdersStatus::name);

// 正例:UNKNOWN 兜底
Converter.ofNullable(String.class, OrdersStatus.class,
    s -> {
        try { return OrdersStatus.valueOf(s); }
        catch (IllegalArgumentException e) {
            log.warn("Unknown OrdersStatus: {}", s);
            return OrdersStatus.UNKNOWN;
        }
    },
    OrdersStatus::name);

class ConverterRegressionTest {
    private final OrderStateConverter states = new OrderStateConverter();
    private final CustomerProfileConverter customers = new CustomerProfileConverter();

    @Test
    void orderStateRejectsUnknownCode() {
        assertThat(states.from(null)).isNull();
        assertThat(states.from(20)).isEqualTo(OrderState.PAID);
        assertThat(states.to(OrderState.SHIPPED)).isEqualTo(30);

        assertThatThrownBy(() -> states.from(99))
            .isInstanceOf(DataTypeException.class)

    @Test
    void profileConverterRejectsTypeDrift() {

            "{\"displayName\":\"Customer C1002\",\"age\":\"old\"}");

        assertThatThrownBy(() -> customers.from(drifted))
            .isInstanceOf(DataTypeException.class)
            .hasMessageContaining("Invalid account.customer JSONB");
    }
}

10.8.3 ExecuteListener 失败观测

public final class ConversionFailureListener extends DefaultExecuteListener {
    private final MeterRegistry mr;

    public ConversionFailureListener(MeterRegistry mr) { this.mr = mr; }

    @Override public void exception(ExecuteContext ctx) {
        Throwable root = ExceptionUtils.getRootCause(ctx.exception());
        if (!(root instanceof DataTypeException)
                && !(ctx.exception() instanceof DataTypeException)) {
            return;
        }
        String hash = DigestUtils.sha256Hex(
            ctx.sql() == null ? "" : ctx.sql()).substring(0, 12);

        mr.counter("jooq.converter.failure",
                "dialect", String.valueOf(ctx.dialect()),
                "sqlHash", hash)
          .increment();

        log.warn("jOOQ conversion failed: dialect={}, sqlHash={}, sql={}",
            ctx.dialect(), hash, ctx.sql(), ctx.exception());
    }
}

设计取舍

取舍 1:Converter 绑定在 DataType 上而非 RecordMapper

转换覆盖查询双向链路(写参数 + 读结果),而 RecordMapper 仅作用于 fetch。该设计使 where(STATUS.eq(enum)) 与 fetch() 都自动转换,业务无需在两端各写一份。

取舍 2:Converter 与 Binding 分离

Converter 专注值转换(简单场景占多数),Binding 控制完整 JDBC 交互(复杂场景)。两层分离避免简单场景被复杂 SPI 拖累。

取舍 3:ForcedType 在 codegen 阶段生效

<forcedType> 在生成阶段写入字段类型,运行时无配置开销。代价是修改后必须重新 codegen,但收益是字段类型在编译期可见。

取舍 4:不强制 NULL 策略

Converter 自身不规定 NULL 行为,由实现决定。这使加密、JSON、枚举等场景可分别选择"NULL 透传 / NULL 抛异常 / NULL 转默认值"。
