第 19 章 · Query Object Model (QOM)

v7 图解版 · 自动从 v6 markdown 生成 · Mermaid 图原样保留

第 18 章揭示了 jOOQ 内部的 QueryPart 表达式树,但树上的每个节点都是不透明的。如何在运行时读取一条 SELECT 的 FROM 子句、如何安全修改 WHERE 条件而不影响原始实例、如何遍历整棵树收集所有引用的表?org.jooq.impl.QOM(Query Object Model) 就是 jOOQ 为这些场景提供的结构化反射 API。

QOM 让 QueryPart 树从"内部 AST"变成"可控的 public 元编程接口":通过 $ 前缀访问器读子节点、通过单参数变异器返回带替换子树的新查询、通过 $replace() 实现递归模式替换。它把第 18 章的 visitor 内核变成可操作的工具。

注意:QOM 相关遍历与替换属于实验性 Model API,且 $traverse() / $replace() 在 jOOQ 3.19 文档中标注为商业版能力(Express / Professional / Enterprise)。

图 19-3 · 不可变变异时序 — 每次变异生成新对象,原对象不变

19.3.1 动态改写示例

var original = dsl
    .select(ORDERS.ID, ORDERS.AMOUNT)
    .from(ORDERS)
    .where(ORDERS.AMOUNT.gt(10))
    .orderBy(ORDERS.ID);

if (original instanceof QOM.Select<?> sel) {
    var modified  = sel.$where(ORDERS.AMOUNT.gt(50)
                                .and(ORDERS.STATUS.eq("ACTIVE")));
    var projected = sel.$select(List.of(ORDERS.ID));
    var paged     = sel.$where(ORDERS.AMOUNT.gt(20))
                       .$limit(DSL.val(100))
                       .$offset(DSL.val(0));

    System.out.println(dsl.render(original));
    System.out.println(dsl.render(modified));
    System.out.println(dsl.render(projected));
    System.out.println(dsl.render(paged));
}

before / after 对照:

-- original
select "orders"."id", "orders"."amount" from "orders"
where "orders"."amount" > 10
order by "orders"."id"

-- modified
select "orders"."id", "orders"."amount" from "orders"
where "orders"."amount" > 50 and "orders"."status" = 'ACTIVE'
order by "orders"."id"

-- projected
select "orders"."id" from "orders"
where "orders"."amount" > 10
order by "orders"."id"

-- paged
select "orders"."id", "orders"."amount" from "orders"
where "orders"."amount" > 20
order by "orders"."id"
limit 100 offset 0

19.3.2 不可变性的价值

并发使用时,QOM 的价值在于"改写不污染原查询"。把构造好的查询作为只读模板,每个线程通过 $where() / $select() 获取带替换子树的新对象。前提:不要在模板中混入可变 SelectQuery API,不要并发执行同一附着运行时状态的查询对象,不要在替换函数中共享可变状态。

19.4.1 收集 Field 引用

图 19-4a · $traverse 收集字段引用 — 遍历 JOIN 树去重收集 5 个 Field
var query = dsl
    .select(CUSTOMERS.NAME, ORDERS.ID)
    .from(CUSTOMERS)
    .join(ORDERS).on(ORDERS.CUSTOMER_ID.eq(CUSTOMERS.ID))
    .where(ORDERS.AMOUNT.gt(10))
    .orderBy(CUSTOMERS.NAME);

Set<Field<?>> allFields = query.$traverse(
    () -> new LinkedHashSet<Field<?>>(),
    (set, part) -> {
        if (part instanceof Field<?> f && !(f instanceof Param<?>)) {
            set.add(f);
        }
        return set;
    }
);

System.out.println("引用了 " + allFields.size() + " 个字段");
allFields.forEach(f -> System.out.println(" → " + f));

期望输出(去重后):

 → "orders"."id"
 → "orders"."customer_id"
 → "customers"."id"
 → "orders"."amount"

19.4.2 统计与搜索

Set<String> tableNames = query.$traverse(
    () -> new LinkedHashSet<String>(),
    (set, part) -> {
        if (part instanceof Table<?> t && !t.getName().isBlank()) {
            set.add(t.getName());
        }
        return set;
    });

boolean hasSubquery = query.$traverse(
    () -> false,
    (found, part) -> found || (part instanceof QOM.Select<?> && part != query));

List<String> conditionTypes = query.$traverse(
    () -> new ArrayList<String>(),
    (list, part) -> {
        if (part instanceof Condition c) list.add(c.getClass().getSimpleName());
        return list;
    });

19.5.1 NOT 简化示例

Condition complex   = DSL.not(ORDERS.CUSTOMER_ID.eq(1));
Condition doubleNot = DSL.not(DSL.not(ORDERS.AMOUNT.gt(10)));

Function<QueryPart, QueryPart> simplify = part -> {
    if (part instanceof QOM.Not n1 && n1.$arg1() instanceof QOM.Not n2) {
        return n2.$arg1();
    }
    if (part instanceof QOM.Not n && n.$arg1() instanceof QOM.Eq<?> eq) {
        return eq.$arg1().ne(eq.$arg2());
    }
    return part;
};

System.out.println(dsl.render((QueryPart) complex.$replace(simplify)));
System.out.println(dsl.render((QueryPart) doubleNot.$replace(simplify)));

before / after:

-- complex after
"orders"."customer_id" <> 1

-- doubleNot before
not (not ("orders"."amount" > 10))
-- doubleNot after
"orders"."amount" > 10

19.5.3 用例:行级安全(RLS)注入

图 19-5b · 行级安全(RLS)注入 — 业务代码无感知,QOM 自动追加 tenant_id 条件
public Query applyRLS(Query query, String tenantId) {
    return (Query) query.$replace(part -> {
        if (part instanceof QOM.Select<?> sel) {
            boolean refs = sel.$from().stream()
                .anyMatch(t -> t.getName().equalsIgnoreCase("orders"));
            if (refs) {
                Condition rls = ORDERS.TENANT_ID.eq(tenantId);
                Condition merged = sel.$where() != null
                    ? sel.$where().and(rls)
                    : rls;
                return sel.$where(merged);
            }
        }
        return part;
    });
}

var query   = dsl.selectFrom(ORDERS).where(ORDERS.STATUS.eq("ACTIVE"));
var secured = applyRLS(query, SecurityContext.getTenantId());

before:

select "orders"."id", "orders"."status", "orders"."tenant_id"
from "orders"
where "orders"."status" = 'ACTIVE'

after:

select "orders"."id", "orders"."status", "orders"."tenant_id"
from "orders"
where "orders"."status" = 'ACTIVE'
  and "orders"."tenant_id" = 'T001'

图 19-6:完整管线

图 19-6 · SQL → QOM → SQL 完整管线 — 解析 / 变换 / 渲染三阶段
String inputSql = """
    SELECT u.name, u.email, o.total
    FROM customers u
    JOIN orders o ON o.customer_id = u.id
    WHERE o.status = 'COMPLETED'
    ORDER BY o.total DESC
    """;

var parsed = dsl.parser().parseSelect(inputSql);

var secured = parsed.$replace(part -> {
    if (part instanceof QOM.Select<?> sel) {
        Condition rls = DSL.field(DSL.name("u", "tenant_id"), String.class)
                           .eq(currentTenantId);
        Condition combined = sel.$where() != null
            ? sel.$where().and(rls)
            : rls;
        return sel.$where(combined);
    }
    return part;
});

String outputSql = dsl.render(secured);
List<Object> binds = ((Query) secured).getBindValues();

19.7.1 适配层封装

public class QueryTransformer {

    public Query applyRLSSafe(Query query, String tenantId) {
        try {
            if (query instanceof QOM.Select<?>) {
                return applyRLS(query, tenantId);
            }
        } catch (Exception e) {
            log.warn("QOM 变换失败,回退原查询", e);
        }
        return query;
    }

    public Set<Field<?>> collectFieldsSafe(Query query) {
        return query.$traverse(
            () -> new LinkedHashSet<Field<?>>(),
            (set, part) -> {
                if (part instanceof Field<?> f && !(f instanceof Param<?>)) {
                    set.add(f);
                }
                return set;
            });
    }
}

使用建议:QOM 最适合基础设施级查询操控(多租户隔离、审计注入、SQL 迁移工具)。普通业务代码用 DSL API 更稳。把 QOM 当作元编程工具,而非日常工具。

设计取舍

取舍 1:$ 前缀分隔命名空间

避免与 DSL 的 select / from / where 冲突。代价是 API 看起来"丑",收益是阅读时立即区分构建与反射。

取舍 2:不可变变异而非原地修改

$where(newCond) 返回新 QueryPart,原对象不变。代价是深层改写需要复制部分子树,收益是查询模板可在多线程下安全复用。

取舍 3:$replace() 反复尝试到稳定

允许多条规则组合,jOOQ 反复应用替换直到树不再变化。代价是要避免 A↔B 循环规则,收益是规则可组合可复用。

取舍 4:QOM 仍标记实验性

jOOQ 保留演进 AST 暴露面的余地。代价是用户必须封装适配层,收益是核心团队可继续优化内部实现。

取舍 5:不与 SQL 优化器深度耦合

QOM 只提供结构化 API,不试图替代数据库优化器。这避免了"jOOQ 重写后 DB 又重写"的双重不可控。
