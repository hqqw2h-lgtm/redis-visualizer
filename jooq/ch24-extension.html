jOOQ 不仅是 SQL 构建器,更是可组合的查询引擎。从 SQL 渲染到 JDBC 执行、从类型映射到代码生成,每个环节都暴露明确的扩展接口。更重要的是:jOOQ 自身大量使用这些接口实现核心功能——隐式连接靠 VisitListener,连接管理靠 ExecuteListener,枚举转换靠 Converter,所有内置函数本质都是 QueryPart。

本章拆解每个扩展点:它是什么、jOOQ 自己怎么用、可以拿它做什么,并附完整可运行代码与生成的 SQL。前置:第 18 章(QueryPart 树模型)、第 23 章(代码到 SQL 全路径追踪)。

24.1 扩展点全景

jOOQ 查询生命周期五阶段:Build → Render → Execute → Fetch → Map。每个阶段对应扩展接口,允许在不修改源码的情况下注入自定义逻辑。

24.2 VisitListener

当 jOOQ 将 QueryPart 树渲染为 SQL 字符串时,深度优先遍历整棵树。VisitListener 让你在每个节点遍历开始 / 结束时插入逻辑——最底层、最灵活的渲染扩展点。

24.2.1 框架自用

内部 Listener 用途

24.2.2 实战:租户表渲染守卫

图 24-2-2 · 租户表渲染守卫 — 多层判定确保 TenantContext 不漏
public class TenantContext {
    private static final ThreadLocal<String> CURRENT = new ThreadLocal<>();
    public static String current()           { return CURRENT.get(); }
    public static void set(String tenantId)  { CURRENT.set(tenantId); }
    public static void clear()               { CURRENT.remove(); }
}

public class TenantVisitListener extends DefaultVisitListener {
    private static final Set<String> TENANT_TABLES =
        Set.of("orders", "documents", "salary");

    @Override
    public void visitStart(VisitContext ctx) {
        if (ctx.queryPart() instanceof Table<?> table) {
            if (TENANT_TABLES.contains(table.getName())
                    && TenantContext.current() == null) {
                throw new DataAccessException(
                    "TenantContext is required when rendering table: "
                    + table.getName());
            }
        }
    }
}

-- 业务代码生成的 SQL 不被 VisitListener 偷偷改写
SELECT * FROM "orders"
 WHERE "orders"."status" = 'ACTIVE'
 ORDER BY "orders"."created_at" DESC

VisitListener 适合观察和替换当前 QueryPart,不适合在任意查询上手工追加 WHERE。生产级行级过滤优先用下一节的 Policy。

24.4.1 慢查询检测器

public class SlowQueryListener extends DefaultExecuteListener {
    private static final Logger log = LoggerFactory.getLogger(SlowQueryListener.class);
    private static final long THRESHOLD_MS = 3000;
    private StopWatch watch;

    @Override public void executeStart(ExecuteContext ctx) { watch = new StopWatch(); }
    @Override public void executeEnd(ExecuteContext ctx) {
        long elapsed = watch.split();
        if (elapsed > THRESHOLD_MS) {
            log.warn("SLOW QUERY [{}ms]: {}", elapsed, ctx.sql());
            if (ctx.params() != null) log.warn("  Params: {}", ctx.params());
        }
    }
}

慢查询日志示例:

WARN  SlowQueryListener - SLOW QUERY [4521ms]:
  SELECT "orders".* FROM "orders" JOIN "order_items" ON ... WHERE "orders"."created_at" > ?
  Params: {1=2024-01-01 00:00:00}

24.4.2 死锁记录与重试

public class DeadlockLogListener extends DefaultExecuteListener {
    private static final Logger log = LoggerFactory.getLogger(DeadlockLogListener.class);

    @Override
    public void exception(ExecuteContext ctx) {
        SQLException e = ctx.sqlException();
        if (e != null && "40P01".equals(e.getSQLState())) {
            log.warn("Deadlock detected, sql={}", ctx.sql());
        }
    }
}

private static final int MAX_RETRIES = 3;

<T> T retryOnDeadlock(Supplier<T> txWork) {
    for (int attempt = 0; ; attempt++) {
        try {
            return txWork.get();
        } catch (DataAccessException e) {
            if (!isDeadlock(e) || attempt >= MAX_RETRIES) throw e;
            sleepBackoff(attempt);
        }
    }
}

Listener 只能记录,不能可靠重放 JDBC Statement。真正的重试包在事务边界外层,让整段业务逻辑重新执行。

24.5.1 自动审计时间戳

public class AuditTimestampListener extends DefaultRecordListener {
    @Override
    public void insertStart(RecordContext ctx) {
        if (ctx.record() instanceof UpdatableRecord<?> rec) {
            LocalDateTime now = LocalDateTime.now();
            Field<LocalDateTime> createdAt = rec.field("created_at", LocalDateTime.class);
            Field<LocalDateTime> updatedAt = rec.field("updated_at", LocalDateTime.class);
            Field<String>        createdBy = rec.field("created_by", String.class);
            if (createdAt != null) rec.set(createdAt, now);
            if (updatedAt != null) rec.set(updatedAt, now);
            if (createdBy != null) rec.set(createdBy, SecurityContext.currentUser());
        }
    }

    @Override
    public void updateStart(RecordContext ctx) {
        if (ctx.record() instanceof UpdatableRecord<?> rec) {
            Field<LocalDateTime> updatedAt = rec.field("updated_at", LocalDateTime.class);
            Field<String>        updatedBy = rec.field("updated_by", String.class);
            if (updatedAt != null) rec.set(updatedAt, LocalDateTime.now());
            if (updatedBy != null) rec.set(updatedBy, SecurityContext.currentUser());
        }
    }
}

24.5.2 软删除

public class SoftDeleteListener extends DefaultRecordListener {
    @Override
    public void deleteStart(RecordContext ctx) {
        if (ctx.record() instanceof UpdatableRecord<?> rec) {
            Field<Boolean> deleted = rec.field("deleted", Boolean.class);
            if (deleted != null) {
                rec.set(deleted, true);
                Field<LocalDateTime> deletedAt = rec.field("deleted_at", LocalDateTime.class);
                if (deletedAt != null) rec.set(deletedAt, LocalDateTime.now());
                rec.update();
                rec.changed(false);
                ctx.exception(new SoftDeleteException());
            }
        }
    }
}

24.6.1 计时与 Prometheus

public class TransactionTimingListener extends DefaultTransactionListener {
    private static final Logger log = LoggerFactory.getLogger(TransactionTimingListener.class);
    private static final ThreadLocal<Long> TX_START = new ThreadLocal<>();
    private static final ThreadLocal<String> TX_ID = new ThreadLocal<>();

    @Override public void beginEnd(TransactionContext ctx) {
        TX_START.set(System.nanoTime());
        TX_ID.set(UUID.randomUUID().toString().substring(0, 8));
    }

    @Override public void commitEnd(TransactionContext ctx) {
        long ms = (System.nanoTime() - TX_START.get()) / 1_000_000;
        log.info("TX [{}] committed in {}ms", TX_ID.get(), ms);
        Metrics.timer("jooq.tx.duration", "result", "commit").record(ms, TimeUnit.MILLISECONDS);
        cleanup();
    }

    @Override public void rollbackEnd(TransactionContext ctx) {
        long ms = (System.nanoTime() - TX_START.get()) / 1_000_000;
        log.warn("TX [{}] rolled back after {}ms", TX_ID.get(), ms);
        Metrics.counter("jooq.tx.rollback").increment();
        cleanup();
    }

    private void cleanup() { TX_START.remove(); TX_ID.remove(); }
}

24.7.1 OrderStatus Converter

public enum OrderStatus { PENDING, CONFIRMED, SHIPPED, DELIVERED, CANCELLED }

public class OrderStatusConverter implements Converter<String, OrderStatus> {
    @Override public OrderStatus from(String dbValue) {
        return dbValue == null ? null : OrderStatus.valueOf(dbValue);
    }
    @Override public String to(OrderStatus userValue) {
        return userValue == null ? null : userValue.name();

    @Override public Class<String> fromType()      { return String.class; }
    @Override public Class<OrderStatus> toType()   { return OrderStatus.class; }
}

24.7.2 加密字段 Binding

public class EncryptedBinding implements Binding<String, String> {
    private static final String SECRET_KEY = System.getenv("ENCRYPTION_KEY");

    private String encrypt(String plain) {
        if (plain == null) return null;
        return Base64.getEncoder().encodeToString(AesCipher.encrypt(plain, SECRET_KEY));
    }
    private String decrypt(String encrypted) {
        if (encrypted == null) return null;
        return AesCipher.decrypt(Base64.getDecoder().decode(encrypted), SECRET_KEY);
    }

    @Override public Converter<String, String> converter() {
        return Converter.of(String.class, String.class, this::decrypt, this::encrypt);
    }
    @Override public void sql(BindingSQLContext<String> ctx) {
        ctx.render().visit(DSL.val(ctx.convert(converter()).value()));
    }
    @Override public void set(BindingSetStatementContext<String> ctx) throws SQLException {
        ctx.statement().setString(ctx.index(), encrypt(ctx.value()));
    }
    @Override public void get(BindingGetResultSetContext<String> ctx) throws SQLException {
        ctx.value(decrypt(ctx.resultSet().getString(ctx.index())));
    }
    @Override public void register(BindingRegisterContext<String> ctx) throws SQLException {
        ctx.statement().registerOutParameter(ctx.index(), Types.VARCHAR);
    }
}

24.8.1 去前缀策略

public class PrefixFreeStrategy extends DefaultGeneratorStrategy {
    @Override
    public String getJavaClassName(Definition def, Mode mode) {
        String name = super.getJavaClassName(def, mode);
        if (name.startsWith("Tbl")) name = name.substring(3);
        if (name.startsWith("V")
            && def instanceof TableDefinition td && td.isView()) {
            name = name.substring(1);
        }
        return name;
    }

    @Override
    public List<String> getJavaClassImplements(Definition def, Mode mode) {
        if (mode == Mode.POJO) return List.of("java.io.Serializable");
        return super.getJavaClassImplements(def, mode);
    }
}

24.8.2 DTO + Lombok

public class DtoLombokStrategy extends DefaultGeneratorStrategy {
    @Override public String getJavaClassName(Definition def, Mode mode) {
        String name = super.getJavaClassName(def, mode);
        return switch (mode) {
            case POJO   -> name + "Dto";
            case RECORD -> name + "Record";
            default     -> name;
        };
    }
    @Override public List<String> getJavaClassAnnotations(Definition def, Mode mode) {
        if (mode == Mode.POJO) return List.of(
            "@lombok.Data", "@lombok.Builder",
            "@lombok.NoArgsConstructor", "@lombok.AllArgsConstructor");
        return super.getJavaClassAnnotations(def, mode);
    }
}

public class N1DetectorListener implements DiagnosticsListener {
    private static final Logger log = LoggerFactory.getLogger(N1DetectorListener.class);

    @Override public void repeatedStatements(DiagnosticsContext ctx) {
        log.error("N+1 DETECTED! Statement executed {}x: SQL: {}",
            ctx.repeatedStatements().size(), ctx.message());
    }
    @Override public void duplicateStatements(DiagnosticsContext ctx) {
        log.warn("DUPLICATE SQL ({}x): {}", ctx.duplicateStatements().size(), ctx.message());
    }
    @Override public void tooManyRows(DiagnosticsContext ctx) {
        if (ctx.resultSetConsumedRows() > 10_000) {
            log.warn("TOO MANY ROWS ({}): {}", ctx.resultSetConsumedRows(), ctx.message());
        }
    }
    @Override public void tooManyColumnsFetched(DiagnosticsContext ctx) {
        log.info("SELECT * detected: {}", ctx.message());
    }
}

图 24-10 · Custom QueryPart 渲染流程 — accept(Context) 双路径 · 3 个便捷基类

框架自用:jOOQ 所有内置函数(DSL.count()、DSL.coalesce()、DSL.trunc() 等)本质都是 QueryPart 实现类。Custom QueryPart 与内置函数享有完全相同的渲染、绑定、转换能力。

24.10.1 PostgreSQL JSONB @>

public class JsonbContainsCondition extends CustomCondition {
    private final Field<JSONB> lhs;
    private final Field<JSONB> rhs;

    public JsonbContainsCondition(Field<JSONB> lhs, Field<JSONB> rhs) {
        this.lhs = lhs;
        this.rhs = rhs;
    }

    @Override
    public void accept(Context<?> ctx) {
        ctx.visit(lhs).sql(" @> ").visit(rhs);
    }
}

public static Condition jsonbContains(Field<JSONB> field, String jsonValue) {
    return new JsonbContainsCondition(field, DSL.val(JSONB.jsonb(jsonValue)));
}

Result<CustomersRecord> vips = dsl.selectFrom(CUSTOMERS)
    .where(jsonbContains(CUSTOMERS.METADATA, "{\"vip\":true}"))
    .and(CUSTOMERS.ACTIVE.isTrue())
    .orderBy(CUSTOMERS.NAME)
    .fetch();

SELECT *
  FROM "customers"
 WHERE "customers"."metadata" @> '{"vip":true}'::jsonb
   AND "customers"."active" = true
 ORDER BY "customers"."name" ASC

24.10.2 自定义聚合 string_agg

public class StringAgg extends CustomField<String> {
    private final Field<String> field;
    private final String separator;
    private final SortField<?>[] orderBy;

    public StringAgg(Field<String> field, String sep, SortField<?>... orderBy) {
        super("string_agg", SQLDataType.VARCHAR);
        this.field = field;
        this.separator = sep;
        this.orderBy = orderBy;
    }

    @Override
    public void accept(Context<?> ctx) {
        ctx.sql("string_agg(").visit(field).sql(", ").visit(DSL.inline(separator));
        if (orderBy.length > 0) {
            ctx.sql(" ORDER BY ");
            for (int i = 0; i < orderBy.length; i++) {
                if (i > 0) ctx.sql(", ");
                ctx.visit(orderBy[i]);
            }
        }
        ctx.sql(")");
    }
}

SELECT "customers"."id",
       string_agg("orders"."status", ', ' ORDER BY "orders"."status" ASC) AS "all_books"
  FROM "customers"
  JOIN "orders" ON "orders"."customer_id" = "customers"."id"
 GROUP BY "customers"."id"

24.13.3 Listener 同时记录 SQL 与 binds

@Test
void boundSqlLogListener_recordsBoth() {
    var probe = new ArrayList<String>();
    DSLContext logged = DSL.using(configuration.derive(
        ExecuteListenerProvider.of(() -> new DefaultExecuteListener() {
            @Override public void renderEnd(ExecuteContext c)  { probe.add("sql:" + c.sql()); }
            @Override public void bindEnd(ExecuteContext c)    { probe.add("binds:" + c.query().getBindValues()); }
        })));

    logged.selectFrom(ORDERS).where(ORDERS.STATUS.eq("ACTIVE")).fetch();

    assertThat(probe).anyMatch(s -> s.startsWith("sql:"));
    assertThat(probe).anyMatch(s -> s.startsWith("binds:[ACTIVE]"));
}

表 24-14:源码入口

入口 路径 关注点

设计取舍

取舍 1:多个窄 SPI 而非单一插件

每个扩展点只看自己的生命周期。代价是要选对扩展点,选错会制造脆弱 hack(如用 VisitListener 做执行重试);收益是边界清晰,风险可定位。

取舍 2:Converter 与 Binding 分层

简单类型映射用 Converter 低成本,协议级绑定用 Binding 深入控制。代价是两个接口共存,新手需要决策;收益是 80% 业务用 Converter,只有加密 / JSONB / 厂商类型才上 Binding。

取舍 3:VisitListener 可观察遍历

对所有查询透明。代价是修改 SQL 结构风险高(别名 / 作用域 / bind),收益是底层灵活。优先用 Policy / QOM / Custom QueryPart。

取舍 4:Custom QueryPart 享受内核

方言、参数、监听器全部复用。代价是必须只依赖 public API,并用 SQL + bind 双断言护航升级;收益是与内置函数同等地位。
